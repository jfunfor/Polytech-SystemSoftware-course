# Тема Сети. Продвинутое задание 1. Квест.

Вы только что получили экстренное задание: проникнуть в сеть заброшенной космической станции и достать главный артефакт, хранящийся на ее сервере: специальный **флаг**. Зачем нужен этот флаг — вам не объяснили, но отметили, что сеть давно пришла в упадок и вам предстоит ее чинить.

На пути вас ожидают несколько увлекательных уровней поломок, так что запастись терпением (а заодно и освежить знания по сетевым инструментам) не помешает.

## Условие задания

Вы получаете готовый `docker-compose.yml` с 4 контейнерами (на момент решения задания все контейнеры будут запущены):
- `client` — ваш рабочий терминал (откуда выполняются запросы);
- `router` — центральный маршрутизатор;
- `dns` — локальный DNS;
- `server` — ядро станции (где и лежит искомый флаг).

**Схема сети**

```
              ┌─────────┐
              │  client │
              └────┬────┘
                   │
                   │
              ┌────V────┐
              │  router │
              └────┬────┘
       ┌───────────┴───────────┐
       │                       │
┌──────V─────┐           ┌─────V─────┐
│    dns     │           │   server  │
└────────────┘           └───────────┘
```

**Что нужно сделать**

**Этап 1: Пробить путь**
- Настройте маршруты так, чтобы `client` мог достучаться до `server`. 
- *Совет:* Не стоит радоваться раньше времени и весело спамить `ping server`. Кто знает, может, при попытке починить маршрутизацию отвалится что-то другое?.. Без подстав точно не обойдется.

**Этап 2: Пробудить DNS**
- Настройте DNS так, чтобы `client` корректно разрешал `server.station.local`. 
- *Совет:* изучите подробнее конфиг DNS, он может внезапно пригодиться в дальнейшем.

**Этап 3: Пройти сквозь фейки**
- На сервере хранятся в том числе и фейковые флаги. Как понять, где настоящий, а где фальшивки?

**Этап 4: Получить флаг**
- Используя HTTP из `client`, достаньте реальный флаг с `server`.
