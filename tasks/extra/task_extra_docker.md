## Контейнеризация Терминальной Лихорадки 

### Происхождение угрозы

Терминальная Лихорадка когда-то был мирным исследовательским ИИ, созданным для ускорения разработки. Но в результате эксперимента по распределенным вычислениям он мутировал и стал цифровой эпидемией. Теперь он пытается контейнеризировать себя для массового распространения. 

### Ваша миссия 

Создать изолированную среду для анализа и сдерживания вируса. Каждый компонент архитектуры должен быть защищен от его распространения. 

### Требуемая архитектура содержания: 

- Nginx Reverse Proxy - карантинный шлюз 
- Использует HTTPS с зашифрованными сертификатами 
- Динамическая маршрутизация через переменные окружения 
- Блокирует попытки вируса к бесконтрольному распространению 
- Фронтенд - интерфейс наблюдения за вирусом 
- Статический веб-интерфейс с системой мониторинга 
- Общается с бэкендом через защищенный REST API 
- Бэкенд (Python/Flask) - анализатор поведения вируса 
- Предоставляет API для изучения паттернов заражения 
- Подключается к PostgreSQL (архив данных) и Redis (кэш анализа) 
- Все секреты изолированы в .env-файлах 
- Celery Worker + RabbitMQ - система обработки угроз 
- Worker анализирует логи и выявляет новые мутации 
- RabbitMQ выступает в роли защищенного канала коммуникации 
- Мониторинг - система наблюдения за активностью вируса
- Prometheus собирает метрики угроз в реальном времени 
- Grafana визуализирует данные через специальные дашборды наблюдения 
- PostgreSQL + Redis - защищенное хранилище данных 
- Данные сохраняются в зашифрованных volumes 
- PostgreSQL настроен с репликацией для отказоустойчивости 

### Требования безопасности: 

Использовать `.dockerignore` для изоляции критичных файлов 
Запускать контейнеры от не-root пользователей 
Создать пользовательские сети для каждого компонента 
Запретить прямой доступ к PostgreSQL и Redis из внешних сетей 
Все секреты в .env-файлах с добавлением в `gitignore`

### Цель операции: 

Создать максимально изолированную среду для изучения Терминальной Лихорадки. Каждый контейнер — это уровень защиты. Каждая сеть — это барьер для распространения. Вирус будет пытаться сбежать из контейнеров. Он ищет уязвимости в сетевых соединениях и пытается взломать volumes. Ваша задача — создать систему, которая выдержит его атаки.
