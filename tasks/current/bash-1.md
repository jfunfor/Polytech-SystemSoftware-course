### Ваша задача:

Написать скрипт `log_analyzer.sh`, который анализирует логи приложения со **строго заданным форматом вывода**.

Формат логов:

Каждая строка в лог-файле `app.log` имеет вид:
```
[YYYY-MM-DD HH:MM:SS] [LEVEL] [MODULE] Сообщение
```

Пример:
```
[2025-03-15 08:12:45] [ERROR] [AUTH] Неверный пароль  
[2025-03-15 08:13:02] [INFO] [DB] Запрос выполнен за 0.2s
```

Допустимые значения:
- Уровни (`LEVEL`): `ERROR`, `WARN`, `INFO`, `DEBUG`;
- Модули (`MODULE`): `AUTH`, `DB`, `NETWORK`, `API`.

### Обязательные аргументы

Скрипт должен принимать аргументы командной строки:
```
./log_analyzer.sh --file <путь_к_файлу> [--level <LEVEL>] [--module <MODULE>] [--stats]
```
- `--file` — путь к файлу логов (обязательный);
- `--level` — фильтр по уровню (`ERROR`/`WARN`/`INFO`/`DEBUG`);
- `--module` — фильтр по модулю (`AUTH`/`DB`/`NETWORK`/`API`);
- `--stats` — вывести статистику (несовместимо с `--level` и `--module`).

Ожидаемое поведение:

Если передан `--level` или `--module`:
- Вывести все записи, соответствующие фильтру.
- Формат вывода:
```
== FILTERED LOGS ==  
<N> entries found  
<запись_1>  
<запись_2>  
...
```

Пример:
```
$ ./log_analyzer.sh --file app.log --level ERROR
== FILTERED LOGS ==
2 entries found
[2025-03-15 08:12:45] [ERROR] [AUTH] Неверный пароль
[2025-03-15 09:30:00] [ERROR] [DB] Connection failed
```

Если передан `--stats`:
- Вывести статистику по уровням и модулям.
- Формат вывода:
```
== STATISTICS ==  
ERROR: <количество>  
WARN: <количество>  
INFO: <количество>  
DEBUG: <количество>  
---  
AUTH: <количество>  
DB: <количество>  
NETWORK: <количество>  
API: <количество>
```

Пример:
```
$ ./log_analyzer.sh --file app.log --stats
== STATISTICS ==
ERROR: 2
WARN: 5
INFO: 10
DEBUG: 1
---
AUTH: 3
DB: 8
NETWORK: 4
API: 3
```

### Обработка ошибок

Если файл не существует или строка не соответствует формату (сообщение можно выбрать произвольное):
```
== ERROR ==  
<сообщение_об_ошибке>
```

Примеры:
```
$ ./log_analyzer.sh --file missing.log
== ERROR ==
File not found: missing.log
$ ./log_analyzer.sh --file bad.log
== ERROR ==
Invalid log format in line 3: "bad line"
```

Технические требования:

1. Каждая строка логов должна соответствовать шаблону:
```
^\[\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2}\] \[(ERROR|WARN|INFO|DEBUG)\] \[(AUTH|DB|NETWORK|API)\] .+$
```
2. При обнаружении некорректной строки — вывести ошибку и завершить работу.
3. Коды возврата:
	- `0` — успешное выполнение;
	- `1` — ошибка аргументов;
	- `2` — ошибка файла/формата.
4. Скрипт необходимо сохранить в директории с заданием.

Файловая структура задания

```
/home/student/bash-1/  
├── app.log                  # файл с логами
└── log_analyzer.sh          # скрипт (решение)
```

### Итог выполнения задания

Обратите внимание на то, что будет проверяться автоматически:

1. Проверка аргументов, корректное поведение скрипта при ошибках файла и формата логов.
2. Фильтрация выбранным методом.
3. Правильный сбор статистики.
4. Форматирование вывода, соответствующее описанному шаблону.
