# Тема Устройства хранения. Задание 3. RAID1-массив.

В мегакорпорации Militech срочно внедряют сервер для осуществления ИИ-контроля боевых дронов. Он должен работать бесперебойно и обрабатывать огромное количество данных. Ваше задание — поднять **RAID1-массив**, настроить **swap** под временные данные и убедиться, что файловая система поддерживает большие таблицы **inode** — для хранения характеристик дронов и логов боевых операций.

## Условие задания

1. Создать RAID1-массив `/dev/md0`, задействовав `/dev/sde` и `/dev/sdf`. RAID должен быть активен после перезагрузки (настройка `mdadm.conf`).
2. Создайте файловую систему `ext4` на RAID. Установите размер `inode` в **256 байт**.
3. Смонтируйте RAID. Точка монтирования: `/mnt/ai_storage`.
4. Создайте swap-файл:
	- Объем: 1GB;
	- Путь: `/swapfile`;
	- Не забудьте включить swap и прописать в `/etc/fstab`.
5. Создайте текстовый файл в `/mnt/ai_storage` с любым содержимым и любым названием.
6. Искусственно выведите из строя диск `/dev/sdf`:
	- Отметьте диск как неисправный (`mdadm /dev/md0 --fail /dev/sdf`);
	- Удалите диск из массива (`mdadm /dev/md0 --remove /dev/sdf`).
7. Проверьте, что файл, созданный в пункте 5, по-прежнему доступен.

**Ожидаемый результат в формате вывода команд:**

```
$ lsblk ...
NAME    MAJ:MIN RM  SIZE RO TYPE  MOUNTPOINT
sde       8:64   0   10G  0 disk  
└─md0     9:0    0   10G  0 raid1 /mnt/ai_storage

$ ...
NAME FSTYPE LABEL UUID                                 MOUNTPOINT
md0  ext4         a1b2c3d4-1234-5678-9101-112131415161 /mnt/ai_storage

$ ...
NAME      TYPE SIZE USED PRIO
/swapfile file   1G   0B   -2

$ ... (до удаления диска из массива)
/dev/md0:
           Raid Level : raid1
           State      : clean 
           Active Devices : 2
           Working Devices : 2
           Failed Devices : 0

$ ... (после удаления диска из массива)
/dev/md0:
           Raid Level : raid1
           State      : degraded 
           Active Devices : 1
           Working Devices : 1
           Failed Devices : 1
```

Это пример, у вас будут другие цифры и UUID.

## Автоматическая проверка 

Обратите внимание на то, что будет проверяться автоматически:

1. Корректность создания и работы RAID1-массива (файловая система массива, размер inode, точка монтирования).
2. Активный swap и корректность `/etc/fstab`.
