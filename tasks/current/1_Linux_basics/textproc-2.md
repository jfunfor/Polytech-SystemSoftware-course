### Ваша задача:

Исходные данные: файлы `manifest.json` и `manifest.yaml`.

Примеры содержимого (файлы генерируются рандомно):
1. `manifest.json`: 
```
[
  {"id": "core.1", "status": "active", "priority": 10},
  {"id": "net.12", "status": "inactive", "priority": 5},
  {"id": "auth.7", "status": "active", "priority": 8},
  ...
]
```

2. `manifest.yaml`:
```
- id: core.1
  status: active
  priority: 10
- id: net.12
  status: inactive
  priority: 5
- id: auth.7
  status: active
  priority: 8
- ...
```

1. Необходимо, используя любой из файлов (`JSON` или `YAML`), отфильтровать активные узлы, где `"status": "active"`.
2. Отсортировать их по полю `priority` **по убыванию**.
3. Вывести результат в шаблонизированном виде в двух вариантах:
	- формат `ID | PRIORITY`;
	- формат Markdown (таблица).
4. Сохранить вывод в файлах `result.txt` и `result.md`.  Указанные файлы должны быть сохранены в директории с заданием.

Примеры выходных файлов:
1. Стандартный (`result.txt`):
```
core.1   | 10
auth.7   | 8
...
```

2. Markdown (`result.md`):
```
| ID     | PRIORITY |
|--------|----------|
| core.1 | 10       |
| auth.7 | 8        |
...
```

Файловая структура задания

```
/home/student/textproc-2/  
├── manifest.json        # исходный файл JSON
├── manifest.yaml        # исходный файл YAML   
├── result.txt           # выходной файл стандарт (решение)
└── result.md            # выходной файл Markdown (решение)
```

### Итог выполнения задания

Обратите внимание на то, что будет проверяться автоматически:

1. Наличие обоих выходных файлов.
2. Соответствие текстового содержимого выходных файлов ожидаемому.
3. Корректность формата файлов, сортировки.
