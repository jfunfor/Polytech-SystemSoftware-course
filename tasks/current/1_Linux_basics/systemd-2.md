## Цербер для вируса

![](../../../pattern/images/dialogs/dialog_lumi_systemd2.png)
*Луми сидит в кресле, закутанный в плед. Металлические пятна на его коже стали менее заметными. Он смотрит на экран с чертежами пакета, и в его глазах загорается знакомый огонёк стратега.*

— Вирус отступил, но он умный! Нам с тобой нельзя расслабляться. Он учится на наших действиях, приспосабливается. Нам нужна система наблюдения, постоянный мониторинг.
Придумал! Создадим Цербера. Не мифического трёхголового пса, а скромного телеметрического сторожа. DEB-пакет, который установит сервис для отслеживания активности системы. Он будет тихо работать в фоне и сообщать в journald о любых подозрительных действиях.
Пакет должен быть идеальным. Автоматически активироваться при установке и бесследно удаляться, если понадобится. Главное, вирус не должен обнаружить нашего сторожа.

![](../../../pattern/images/bars/bar_systemd2.png)

### Ваша задача:

Создать `.deb`-пакет для сервиса `cerberus-telemetry`, который:
1. Размещает скрипт `cerberus-log.sh` в `/usr/bin/`.
2. Устанавливает `systemd`-сервис (`cerberus.service`), который:
    - Запускается при загрузке системы (`multi-user.target`);
    - Пишет в `journald` сообщения с тегом `cerberus`.
3. Автоматически активирует сервис после установки.
4. Корректно удаляет все файлы и останавливает сервис при деинсталляции.

Требуемая структура пакета

```
cerberus-telemetry/
├── DEBIAN/                       
│   ├── control                   # метаданные пакета
│   └── postinst, postrm          # скрипты установки/удаления
├── usr/
│   └── bin/cerberus-log.sh       # bash-скрипт
├── lib/systemd/system/
│   └── cerberus.service          # systemd-сервис
```

Файловая структура задания

```
/home/student/systemd-2/  
└── cerberus-log.sh               # bash-скрипт для пакета (не изменяйте)
```

### Итог выполнения задания

Обратите внимание на то, что будет проверяться автоматически:

1. Корректная установка пакета.
2. Включение и запуск сервиса после установки.
3. Проверка логов.
4. Остановка и отключение сервиса после удаления пакета.
