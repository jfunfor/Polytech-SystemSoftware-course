# Тема Сети. Задание 1. Призрак в сетях.

Мегакорпорации контролируют почти все аспекты жизни, включая доступ к сети. Вы — сетевой инженер-революционер по прозвищу _NetGhost_, работающий на подпольную группировку. Ваша цель — перехватить и проанализировать сетевой трафик корпорации **NeuroNet**, чтобы раскрыть правду об их ИИ-экспериментах.

У вас есть доступ к терминалу в их дата-центре. Однако у вас нет интерфейса с графикой: только консоль, командная строка и голый Linux.

## Условие задания

Откройте файл `task.json`, расположенный в директории задания, и ознакомьтесь с вашими параметрами (генерируются рандомно).

Пример файла `task.json`:
```
{
  "domain": "DOMAIN",
  "port": PORT,
  "dns_type": "TYPE",
  "hop": HOP,
  "tasks": {
    "ping": "Определить процент успешных ответов от $DOMAIN (из 5 попыток).",
    "traceroute": "Найти IP $HOP-го узла в маршруте к $DOMAIN.",
    "dns": "Получить $TYPE-запись для $DOMAIN.",
    "port_scan": "Проверить, открыт ли порт $PORT на $DOMAIN.",
    "listen_ports": "Подсчитать количество LISTEN-портов на машине."
  }
}
```

Параметры `DOMAIN` (уникальный домен, например, `example.com`), `PORT` (уникальный номер порта, например, `80`), `HOP` (уникальный номер узла на маршруте, например, `3`) и `TYPE` (уникальный тип записи, например, `А`) будут иметь конкретные значения.

Результат выполнения задания необходимо сохранить в файле `result.json`, расположенном в директории с заданием. В файле уже есть шаблон, который необходимо заполнить вручную или с помощью скрипта.

Шаблон `result.json` c возможными данными:
```
{
  "ping_loss": "0",
  "traceroute_hop_n": "192.168.1.1",
  "dns_result": "140.82.121.3",
  "port_scan": "open/closed",
  "listen_ports": "15"
}
```

**Необходимо:**
1. Проверить доступность домена `DOMAIN`:
	- Что сделать: Определить **процент потерь пакетов** (например, `0%` если все ответы получены);
	- Куда сохранить: В поле `ping_loss` файла `result.json`.
2. Проанализировать маршрут до `DOMAIN`:
	- Что сделать: Найти IP-адрес `HOP`-го  узла (хоста) в маршруте;
	- Куда сохранить: В поле `traceroute_hop_n` файла `result.json`.
3. Найти DNS-запись `DOMAIN`:
	- Что сделать: Получить значение указанной DNS-записи (для `A` — IP, для `MX` — почтовый сервер);
	- Куда сохранить: В поле `dns_result` файла `result.json`.
4. Проверить состояние порта `PORT` на `DOMAIN`:
	- Что сделать: Определить, открыт ли порт (`open`/`closed`);
	- Куда сохранить: В поле `port_scan` файла `result.json`.
5. Выполнить анализ локальных портов:
	- Что сделать: Подсчитать количество портов в состоянии `LISTEN`;
	- Куда сохранить: В поле `listen_ports` файла `result.json`.

**Внимание:** заполняйте файл `result.json` строго по шаблону.
