# Лекция 23 - Базовые запросы и фильтрация данных

В прошлой лекции мы спроектировали структуру базы данных, в этой лекции мы переходим к общению с базой данных, используя выражение `SELECT`.

## Принцип выполнения SQL-запросов

Прежде чем переходить к синтаксису `SELECT`, важно понимать, что происходит, когда мы отправляем запрос в базу данных.

Первым делом клиент устанавливает соединение с сервером PostgreSQL, передавая имя пользователя и пароль. Сервер создаёт сессию с уникальным идентификатором. После создания сессии, можно выполнять запросы. При каждом запросе перед выполнением сервер проверяет: корректность синтаксиса запроса, есть ли у пользователя доступ к данным и на выполнение выражений.

Если запрос прошёл проверку, то дальше он передаётся оптимизатору, который определяет наиболее эффективный способ выполнения запроса. Он рассматривает порядок соединения таблиц, доступные индексы, а затем определяет план выполнения запроса. После выполнения запроса согласно плану сервер возвращает **результирующий набор**.

## Блоки запроса

Выражения `SELECT` как правило состоят из нескольких компонентов - блоков (clauses). Обязательным является только один из них - `SELECT`, однако, как правило, его одного недостаточно. Каждый блок отвечает за определенный этап обработки данных.

### SELECT

Блок `SELECT` определяет, какие именно столбцы попадут в результирующий набор запроса. Кроме определения самих столбцов из таблиц также можно в блоке `SELECT` указывать: литералы (строки и числа), выражения и вызовы встроенных функций, например `ROUND()`.

Рассмотрим запрос:
```
SELECT student_id,
	'ACTIVE' AS status,
	UPPER(first_name)
FROM students;

 student_id | status |  upper
------------+--------+---------
          1 | ACTIVE | ИВАН
          2 | ACTIVE | МАРИЯ
          3 | ACTIVE | АЛЕКСЕЙ
          4 | ACTIVE | ЕЛЕНА
(4 строки)
```

В результирующий набор попал первый столбец из таблицы `students`, литерал `ACTIVE` и результат встроенной функции `UPPER()`. Более того, данный пример демонстрирует определение псевдонима для столбца с использованием ключевого слова `AS`. Использование ключевого слова необязательно, но для повышения читаемости кода его использование приветствуется.

Для уничтожения дублирующихся строк используется ключевое слово `DISTINCT`. Пример использования:
```
SELECT DISTINCT group_id
FROM students;

 group_id
----------
        3
        2
        1
(3 строки)
```

Как вы думаете что произойдёт, если использовать `DISTINCT` для двух или более колонок? Будут удаляться записи, которые имеют одинаковые значения во всех колонках:
```
SELECT DISTINCT group_id, student_id
FROM students;

 group_id | student_id
----------+------------
        1 |          1
        1 |          5
        2 |         12
        1 |          6
...

(19 строк)
```

Если не требуется, чтобы сервер удалял дублирующиеся строки, то вместо `DISTINCT` можно указать `ALL`, однако `ALL` применяется по умолчанию.

> Хотя SELECT пишется в начале, выполняется он одним из последних.

### FROM

Блок `FROM` определяет таблицы, используемые в запросе, а также способ их связывания.

В блоке `FROM` можно указывать следующие типы таблиц:
- Постоянные таблицы, созданные с помощью `CREATE TABLE`.
- Временные таблицы, возвращаемые подзапросом.
- Виртуальные таблицы (представления).

На данный момент мы ограничимся постоянными таблицами, но по мере изучения SQL мы познакомимся и с другими. Связывание таблиц также будет рассмотрено позже, однако мы с ним уже встречались в конце прошлой лекции.

Также в блоке `FROM` можно задавать псевдонимы для таблиц, что очень удобно при использовании нескольких таблиц. При определении псевдонима таблицы столбцы больше не доступны по её оригинальному названию, и в запросе необходимо использовать именно псевдоним. Псевдоним задаётся аналогично столбцам с использованием ключевого слова `AS` (или без него, т. к. он не обязателен):
```
SELECT g.group_name, s.student_id, s.average_grade
FROM students AS s
LEFT JOIN groups AS g ON s.group_id = g.group_id;

 group_name | student_id | average_grade
------------+------------+---------------
 УКФС-01-23 |          5 |          4.50
 УКФС-01-23 |          6 |          4.80
 УКФС-01-23 |          7 |          3.90
 УКФС-01-23 |          8 |          5.00
...
(19 строк)
```

### WHERE

Блок `WHERE` отсеивает нежелательные строки из результирующего набора.

Например, мы хотим переделать предыдущий запрос, чтобы в результирующий набор попали студенты со средним баллом выше 4.5, тогда нам необходимо добавить блок `WHERE`:
```
SELECT g.group_name, s.student_id, s.average_grade
FROM students AS s
LEFT JOIN groups AS g ON s.group_id = g.group_id
WHERE s.average_grade > 4.5;

 group_name | student_id | average_grade
------------+------------+---------------
 УКФС-01-23 |          6 |          4.80
 УКФС-01-23 |          8 |          5.00
 ПИ-33-22   |         12 |          4.60
 ПИ-33-22   |         13 |          4.90
 ТИИ-02-21  |         18 |          4.70
 УКФС-01-23 |          2 |          4.90
 ТИИ-02-21  |          4 |          4.60
(7 строк)
```

Также мы можем добавить условия фильтрации, используя `OR` и `AND`:
```
SELECT g.group_name, s.student_id, s.average_grade
FROM students AS s
LEFT JOIN groups AS g ON s.group_id = g.group_id
WHERE s.average_grade > 4.5
	AND g.group_id = 1;

 group_name | student_id | average_grade
------------+------------+---------------
 УКФС-01-23 |          6 |          4.80
 УКФС-01-23 |          8 |          5.00
 УКФС-01-23 |          2 |          4.90
(3 строки)
```

### GROUP BY

Блок `GROUP BY` группирует строки по одинаковым значениям столбцов. Обычно данный блок используется вместе с агрегатными функциями, например `COUNT()`:
```
SELECT g.group_name, COUNT(*) AS stud_cnt
FROM students AS s
LEFT JOIN groups AS g ON s.group_id = g.group_id
WHERE s.average_grade > 4.5
GROUP BY g.group_id;

 group_name | stud_cnt
------------+----------
 ТИИ-02-21  |        2
 ПИ-33-22   |        2
 УКФС-01-23 |        3
(3 строки)
```

В данном примере мы подсчитываем количество студентов со средним баллом выше 4.5 в каждой группе.

### HAVING

Можно было заметить в предыдущем примере, что `WHERE` применяется до группировки, а что, если нужно отфильтровать данные и после группировки? На помощь приходит блок `HAVING`. Блок `HAVING` ограничивает число строк в результирующем наборе после группировки данных.

Дополним предыдущий запрос, чтобы вывести группы, в которых набралось хотя бы 3 таких студента:
```
SELECT g.group_name, COUNT(*) AS stud_cnt
FROM students AS s
LEFT JOIN groups AS g ON s.group_id = g.group_id
WHERE s.average_grade > 4.5
GROUP BY g.group_id
HAVING COUNT(*) >= 3;

 group_name | stud_cnt
------------+----------
 УКФС-01-23 |        3
(1 строка)
```

Обратите внимание, что в `HAVING` мы не можем указать `stud_cnt`, так как блок `SELECT` выполняется после `HAVING`.

### ORDER BY и LIMIT

Блок `ORDER BY` сортирует результирующий набор на основании данных в столбцах или выражения. При сортировке можно указать порядок: по возрастанию (`ASCENDING` или просто `ASC`, используется по умолчанию) и по убыванию (`DESCENDING` или `DESC`). Сортировать также можно по нескольким полям. Рассмотрим пример:
```
SELECT group_id, student_id
FROM students
ORDER BY group_id, student_id;

 group_id | student_id
----------+------------
        1 |          1
        1 |          2
        1 |          5
        1 |          6
        1 |          7
        1 |          8
        1 |          9
        2 |          3
        2 |         10
        2 |         11
        2 |         12
        2 |         13
        2 |         14
        3 |          4
        3 |         15
        3 |         16
        3 |         17
        3 |         18
        3 |         19
(19 строк)
```

В данном случае мы сортируем сначала по id группы, а после по id студентов.

`LIMIT` позволяет извлечь первые N записей, использование с ключевым словом `OFFSET` помогает задать также смещение для `LIMIT` (пропустить первые записи). Примеры:
```
SELECT group_id, student_id
FROM students
ORDER BY group_id, student_id
LIMIT 3;

 group_id | student_id
----------+------------
        1 |          1
        1 |          2
        1 |          5
(3 строки)

SELECT group_id, student_id
FROM students
ORDER BY group_id, student_id
LIMIT 3 OFFSET 3;

 group_id | student_id
----------+------------
        1 |          6
        1 |          7
        1 |          8
(3 строки)
```

### Порядок написания и порядок исполнения блоков

Важно понимать при работе с SQL, что порядок написания не совпадает с порядком выполнения блоков.

Порядок написания:
1. `SELECT`;
2. `FROM`;
3. `WHERE`;
4. `GROUP BY`;
5. `HAVING`;
6. `ORDER BY`;
7. `LIMIT`.

Порядок выполнения:
1. `FROM`;
2. `WHERE`;
3. `GROUP BY`;
4. `HAVING`;
5. `SELECT`;
6. `ORDER BY`;
7. `LIMIT`.

Особенно важно понимать порядок выполнения при использовании псевдонимов, например, в блоке `WHERE` нельзя ссылаться на псевдоним, указанный в `SELECT`.

## Фильтрация

Любое условие в `WHERE` - это выражение, которое возвращает булево значение. В результирующий набор попадают те строки, для которых условие вернуло `TRUE`.

Для построения сложных условий мы используем логические операторы `AND`, `OR` и `NOT`. Важно помнить о приоритетах операций: сначала вычисляется `NOT`, затем `AND` и только потом `OR`. Также можно использовать круглые скобки `()` для приоритезации.

Существует множество способов фильтрации данных, чтобы исключить те или иные данные. Рассмотрим основные типы условий.

### Условия равенства

Самыми простыми условиями являются условия равенства и неравенства. Условие равенства имеет форму: `столбец = выражение`, а условие неравенства: `столбец != выражение` или `столбец <> выражение`.

Примеры запросов с условиями равенства и неравенства:
```
SELECT *
FROM students
WHERE group_id = 1;

SELECT *
FROM students
WHERE group_id != 1;
```

### Условия вхождения в диапазон

Данный тип условий реализуется с использованием операторов сравнения: `>`, `<`, `>=`, `<=`, также можно использовать оператор `BETWEEN`, который проверяет вхождение в диапазон между верхней и нижней границей. `BETWEEN` включает границы. Примеры использования условий вхождения в диапазон:
```
SELECT *
FROM students AS s
LEFT JOIN groups AS g ON s.group_id = g.group_id
WHERE enrollment_year > 2021
	AND enrollment_year < 2023;

SELECT *
FROM students AS s
LEFT JOIN groups AS g ON s.group_id = g.group_id
WHERE enrollment_year BETWEEN 2022 AND 2022;
```

Поскольку строки сравниваются по алфавиту, то `BETWEEN` работает и для них:
```
SELECT s.student_id, s.last_name
FROM students AS s
LEFT JOIN groups AS g ON s.group_id = g.group_id
WHERE SUBSTRING(last_name FOR 1) BETWEEN 'А' AND 'Д';

 student_id | last_name
------------+-----------
          7 | Волков
         15 | Алексеев
          4 | Васильева
(3 строки)
```

В фильтрации данного запроса выделяется первая буква фамилии, используя `SUBSTRING()`, и проверяется её вхождение в диапазон с помощью `BETWEEN`.

### Условия членства

Оператор `IN` проверяет, совпадает ли значение с любым элементом из списка. По сути он является удобной заменой цепочки `OR`. `NOT IN` исключает строки, попадающие в список.

Пример:
```
SELECT s.student_id, g.group_name
FROM students AS s
LEFT JOIN groups AS g ON s.group_id = g.group_id
WHERE g.group_name IN ('УКФС-01-23', 'ПИ-33-22')
LIMIT 5;

 student_id | group_name
------------+------------
          5 | УКФС-01-23
          6 | УКФС-01-23
          7 | УКФС-01-23
          8 | УКФС-01-23
          9 | УКФС-01-23
(5 строк)
```

### Условия соответствия шаблону

Оператор `LIKE` позволяет искать строки по шаблону, используя два специальных символа маски:
- `_` - один любой символ;
- `%` - любая последовательность символов (в том числе ни одного).

Допустим мы хотим найти студентов чьё имя начинается на букву `a`, тогда мы могли бы использовать следующий запрос:
```
SELECT student_id, first_name
FROM students
WHERE first_name LIKE 'a%';

 student_id | first_name
------------+------------
(0 строк)
```

Результатом является пустое множество, дело в том, что `LIKE` чувствителен к регистру. Нужно изменить шаблон и указать правильный регистр. PostgreSQL также предлагает оператор `ILIKE`, который нечувствителен к регистрам:
```
SELECT student_id, first_name
FROM students
WHERE first_name ILIKE 'а%';

 student_id | first_name
------------+------------
          6 | Анна
         11 | Андрей
         13 | Артем
         16 | Алина
          3 | Алексей
(5 строк)
```

Символьные маски хороши для простых выражений поиска. Если же символьные маски не обеспечивают достаточной гибкости, то можно использовать регулярные выражения. В PostgreSQL для использования регулярных выражений используются операторы:
- `~` - строка соответствует регулярному выражению с учетом регистра;
- `~*` - строка соответствует регулярному выражению без учета регистра;
- `!~` - строка не соответствует регулярному выражению с учетом регистра;
- `!~*` - строка не соответствует регулярному выражению без учета регистра.

Пример поиска по регулярному выражению:
```
SELECT student_id, first_name
FROM students
WHERE first_name ~* 'кс';

 student_id | first_name
------------+------------
         15 | Максим
          3 | Алексей
(2 строки)
```

В данном случае мы ищем имена, содержащие `кс`.

### NULL

`NULL` в SQL обозначает отсутствие информации. Это не ноль и не пустая строка. `NULL` не равен ничему, даже другому `NULL`. Операторы сравнения с `NULL` возвращают также `NULL`. `NULL` интерпретируется как `FALSE` в `WHERE`.

Для проверки на `NULL` необходимо использовать специальные операторы:
- `IS NULL` - проверяет, является ли значение `NULL`.
- `IS NOT NULL` - проверяет, что значение известно (не является `NULL`).

```
SELECT NULL = NULL;
 ?column?
----------

(1 строка)

SELECT NULL IS NULL;
 ?column?
----------
 t
(1 строка)

SELECT NULL IS NOT NULL;
 ?column?
----------
 f
(1 строка)
```

В первом случае мы проверяем, равен ли `NULL` другому `NULL`, и результатом является `NULL`, во втором случае мы проверяем, является ли `NULL` неизвестным значением, и получаем `TRUE` (`t`), в третьем случае мы проверяем, является ли `NULL` известным значением, и получаем `FALSE` (`f`).

Также полезной функцией для работы с `NULL` является `COALESCE`, которая принимает список значений и возвращает первоей непустое значение:
```
SELECT COALESCE(NULL, 'Нет данных', 'Есть данные');
  coalesce
------------
 Нет данных
(1 строка)
```
