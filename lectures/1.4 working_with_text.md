# Лекция 4 - Работа с текстом

Говоря о преимуществах работы в командной строке, сложно переоценить вклад инструментов работы с текстом. Они упрощают анализ файлов и данных, преобразование данных между разными форматами, фильтрацию и многое другое.

Для удобства работы с текстовыми данными необходимо правильно понимать, в чем именно заключается задача и уметь правильно ее декомпозировать. Этот навык приобретается исключительно при непосредственной работе с текстом, так что не забудьте потренироваться после прочтения лекции. Также потрудитесь почитать man'ы используемых команд, чтобы осознать всю широту возможностей, которые они вам предоставляют.

# Let's get started!

Начнем с простого: создадим тестовый файл, над которым будем производить все манипуляции. Конечно, мы можем использовать текстовый редактор для создания файла (например, vim, nano или, не дай Бог, emacs), но я предлагаю сделать это более "линуксово":

```sh
echo -e "Cat says moo\nCow says meow" > file.txt
```

Флаг `-e` используется для включения поддержки escape-последовательностей. Также, как видно, был использован оператор перенаправления вывода `>`, который, в данном случае, перезаписывает существующий файл или создает новый.

Можем убедиться в том, что желаемый результат достигнут:

```sh
$ cat file.txt
Cat says moo
Cow says meow
```

Далее будем работать, за редкими исключениями, именно с этим файлом!

# `sed`, Великий и Ужасный

sed - это потоковый текстовый редактор, который, в отличие от vim и nano, не позволяет интерактивно редактировать текс, а лишь применяет заданные правила к его содержимому

Использование: `sed <options> <file>`

sed не меняет данные в обрабатываемом файле, весь его вывод размещается в stdout.

Например:
```sh
$ echo "cat says moo" | sed 's/moo/meow/'
cat says meow
```

Тут мы используем такую функцию bash, как конвеер. Он перенаправил результат echo на вход sed. Подробнее об этом вы можете и даже должны почитать в сети Интернет!

Вернемся к опциям sed. Разберем их структуру:

- `s` - (от англ. substitute) команда замены
- `/moo/meow/` - два "паттерна", которые принимает в качестве аргументов `s`. Нетрудно догадаться, что первый паттерн заменяется на второй. По умолчанию, после замены первого вхождения в каждой строке, sed автоматически переходит к следующей. Далее рассмотрим, как это исправить.

Идем далее. Обработаем файл, задействуя несколько команд. Для этого добавим флаг `-e` при вызове:

```sh
# Сначала рассмотрим пример с файлом. Непорядок, корова не должна мяукать
$ sed 's/moo/meow/' file.txt
Cat says meow
Cow says meow
```

Рассмотрим простой пример: заменим "Cow" на "Cat"
```sh
# Тут все лучше, но корову тоже забывать не следует
$ sed -e 's/moo/meow/; s/Cow/Cat/' file.txt
Cat says meow
Cat says meow
```

```sh
$ sed -e 's/moo/meow/; s/meow/moo/' file.txt
Cat says moo
Cow says moo
```

Непорядок! Тут мы столкнулись с реалиями работы с sed, где каждая команда последовательно запустилаась на каждую строчку файла. Оставим решение этой задачи напоследок. Особо любознательным можно попробовать решить самостоятельно без использования ИИ чат-ботов!

## Флаги sed
Продолжим разбирать работу sed на примере команды замены. Затронем тему флагов, которые позволяют еще сильнее расширить функционал команды.

Разберем пример:

```sh
$ echo "cat says moo moo moo" | sed 's/moo/meow/'
cat says meow moo moo
```

Как видим, заменилось только первое вхождение в строке! Заменим все, используя флаг "g":

```sh
echo "cat says moo moo moo" | sed 's/moo/meow/g'
cat says meow meow meow
```

Мы также можем использовать флаг "w" для записи результатов обработки в файл:

```sh
echo "cat says moo moo moo" | sed 's/moo/meow/w output.txt'
```

Конечно, в файл будут записаны только те строки, к которым была применена команда. Полезно, когда надо отдельно просмотреть результаты работы.

Обо всех флагах можно узнать, прочитав man по утилите sed:

```sh
$ man sed
```

Уверяю вас, вы удивитесь насколько это мощный инструмент!

## Выбор фрагмента текста для обработки

Например, так можно обработать строку с конкретным номером. Нумерация начинается с единицы, как в массивах MATLAB, на радость некоторым преподавателям!

```sh
$ sed '2s/says/humbles/' file.txt
Cat says moo
Cow humbles meow
```

Красота! Рассмотрим еще несколько примеров, менее подробно:

```sh
# Диапазон строк 2-10
$ sed '2,10s/says/humbles/' file.txt
```

```sh
# От второй строки и до конца файла
$ sed '2,$s/says/humbles/' file.txt
```

Еще мы можем фильтровать строки, которые будут подвержены обработки. Это один из способов решить тяжелую ситуацию с двойной заменой meow на moo и обратно. Давайте предположим, что в одной строке не могут одновременно встречаться слова "Cat" и "Cow". В таком случае, можно сделать следующее:

```sh
$ sed -e '/Cat/s/moo/meow/; /Cow/s/meow/moo/;' file.txt
Cat says meow
Cow says moo
```

Конечно, это не полное решение данной задачи, оно работает только при вышеописанном условии. Зато мы научились фильтровать строки для обработки :) Мудрецы утверждают, что вместо таких простых фильтров можно использовать регулярные выражения. Советую проверить самостоятельно!

Разберем еще одну команду - `d` (от "delete"):

Сразу применим фильтрацию и не забудем про замену!

```sh
# Строчка, содержащая слово "Cat" была удалена
$ sed -e '/Cat/d; s/meow/moo/;' file.txt
Cow says moo
```

Также можно указывать два шаблона для фильтрации. В таком случае будут удалены как строки, содержащие перечисленные шаблоны, так и строки, находящиеся между ними:

```sh
echo -e "Cat says moo\nверните\n2007\nCow says meow" | sed '/Cat/,/Cow/d'
# Вывод пустой!
```

Ликвидировав на этот раз и корову, и кота, перейдем к другим занимательным функциям `sed`

# Другие команды

- `i` - добавляет новую строку перед заданной
- `a` - добавляет новую строку после заданной
- `c` - целиком заменяет заданную строку на новую
- `y` - заменяет символы
- `=` - выводит номера строк (в комбинации с ключом n и шаблоном дает вывод номеров строк, соответствующих шаблону)

Внимание: тут слово "заданная" означает заданная как через номер (диапазон), так и с использованием шаблонов фильтрации

На этом разрешите откланяться! Важно понимать, что обработка текстов не ограничивается sed: также важно иметь представление о языке awk, регулярных выражениях и не забывать тренироваться на практике!
