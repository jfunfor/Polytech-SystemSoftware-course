# Лекция 22 - Введение в базы данных

База данных (БД) - это организованное хранилище взаимосвязанных данных. Примером базы данных может служить даже бумажный телефонный справочник.

Для работы с электронной базой данных необходим специальный программный комплекс - система управления базами данных (СУБД). СУБД - это совокупность программных и языковых средств, обеспечивающих управление созданием и использованием баз данных.

Популярные примеры СУБД:
- MySQL, PostgreSQL, Oracle Database, SQLite - реляционные;
- MongoDB - документоориентированная;
- Redis - хранилище "ключ-значение";

СУБД можно классифицировать по модели данных, которая определяет способ организации и хранения информации. Рассмотрим основные типы:

1. **Реляционные**. В основе таких баз данных лежит реляционная модель, в которой данные структурированы в виде набора таблиц. Каждая строка таблицы представляет собой набор связанных значений, которые относятся к одной и той же сущности. Для навигации по взаимосвязанным сущностям используются избыточные данные, связывающие записи разных таблиц. Более подробно про реляционные БД будет позже, так как именно на них мы сконцентрируем наше внимание.
2. **Иерархические**. В этом случае данные организованы в виде древовидной структуры.
3. **Сетевые**. Данные представляют собой набор записей и набор связей, определяющих отношения между разными записями.
4. **Key-value**. Данные хранятся в виде совокупности пар "ключ-значение", в которых ключ служит уникальным идентификатором.
5. **Документоориентированные**. В данном случае данные хранятся в виде документов (обычно в форматах JSON и BSON), сгруппированных по коллекциям. Такие СУБД подходят для полуструктурированных данных, где структура может быть гибкой.

## Реляционная модель

Как мы уже установили, реляционная модель организует данные в виде набора взаимосвязанных таблиц.

### Основные понятия и компоненты таблицы

- Сущность - объект или явление, которое представляет интерес для пользователей базы данных.
- Столбец (атрибут, поле) - отдельный элемент базы данных, хранящийся в таблице. Определяет отдельное свойство сущности. Каждый столбец имеет строго определенный тип данных.
- Строка (запись) - набор значений столбцов, которые вместе описывают сущность.
- Первичный ключ - один или более столбцов, которые уникально идентифицируют строки таблицы.
- Внешний ключ - один или более столбцов, которые можно использовать для идентификации строки другой таблицы по первичному ключу. Внешний ключ устанавливает логическую связь между таблицами, реализуя отношения между сущностями.
  - Таблица, на которую ссылается внешний ключ, называется **родительской** или главной.
  - Таблица, содержащая внешний ключ, называется **дочерней** или зависимой.

> Хотя проектирование грамотной базы данных почти всегда подразумевает использование ключей, наличие первичного и внешнего ключей не является *обязательным* для создания таблицы.

### Связи между таблицами

В реляционной модели SQL существует три типа связей между таблицами:
- **Один к одному (1:1)** - одна запись в одной таблице связана ровно с одной записью в другой.
- **Один ко многим (1:N)** - одна запись в родительской таблице связана с несколькими записями в дочерней.
- **Многие ко многим (M:N)** - многие записи одной таблицы могут быть связаны со многими записями другой. Эта связь реализуется не напрямую, а через промежуточную таблицу-связку, которая содержит внешние ключи на обе таблицы.

### Результат работы с данными

Результатом любого запроса на выборку или соединение таблиц также является таблица. Эта временная, непостоянная таблица называется **результирующий набор**. Даже если в результате запроса возвращается одно значение или пустой список, с точки зрения модели это всё равно таблица (соответствующего размера).

## SQL и классификация команд

SQL - язык структурированных запросов, предназначенный для управления реляционными базами данных. В SQL мы указываем, что нужно сделать с данными, в то время как СУБД сама определяет, как наиболее эффективно выполнить эту операцию.

Всё множество команд в SQL принято делить на четыре группы в зависимости от их функционального назначения:
1. **DDL** (Data Definition Language) - язык определения данных. Команды этой группы используются для создания, изменения и удаления структуры базы данных и её объектов (таблиц, индексов и представлений). При проектировании и построении базы данных или при изменении структуры базы данных, мы используем именно эти команды. Примеры команд: `CREATE`, `ALTER`, `DROP`.
> Схема - совокупность объектов базы данных: таблиц, столбцов, типов данных, ключей, индексов и отношений между таблицами.
2. **DML** (Data Manipulation Language) - язык манипулирования данными. DML отвечает за наполнение базы данных, изменение и выборку информации. Если DDL отвечает на вопрос "как хранить?", то DML отвечает на вопрос "что хранить?". Примеры операторов: `SELECT`, `INSERT`, `UPDATE`, `DELETE`.
3. **DCL** (Data Control Language) - язык управления доступом. Эти команды определяют права доступа пользователей к объектам базы данных. С их помощью можно предоставлять доступ или ограничивать его пользователям для чтения, изменения данных или создания новых объектов. Примеры команд: `GRANT`, `REVOKE`.
4. **TCL** (Transaction Control Language) - язык управления транзакциями. Задачей TCL является контролировать, как фиксируются или отменяются изменения в базе данных. Примеры команд: `BEGIN`, `COMMIT`, `ROLLBACK`.
> Транзакции - это последовательность операций с данными, которые должны быть выполнены как единое целое. Это основа обеспечения целостности данных при сбоях или параллельной работе пользователей.

## Типы данных

В SQL каждый столбец в таблице должен иметь определённый тип данных, указывающий на то, какая информация может храниться в этом столбце.

Основные типы данных:
- Символьные данные.
- Числовые данные.
- Булевые данные.
- Временные данные.

### Символьные данные

Символьные данные могут храниться как строки фиксированной или переменной длины. Строки фиксированной длины дополняются пробелами справа, а строки переменной длины - нет. При определении столбца символьного типа необходимо задать максимальный размер строки. Для описания строк фиксированной длины используется `CHAR(N)`, а для строк переменной длины `VARCHAR(N)`, где N - максимальный размер строки. Если длина `N` указана, то она должна быть больше нуля и не может превышать 10 485 760 (10 МБ). Если `VARCHAR` используется без указания длины `N`, тип принимает строки любой длины. Если `CHAR` не содержит указания длины `N`, это эквивалентно character(1).

PostgreSQL предоставляет `TEXT` тип данных, который позволяет хранить строки любой длины. Как можно было заметить `VARCHAR` без указания `N` аналогичен `TEXT`.

### Числовые данные

Числовые типы включают в себя двух-, четырех- и восьмибайтовые целые числа, четырех- и восьмибайтовые числа с плавающей запятой. Ниже приведены числовые типы и их описание:

| Имя | Размер | Описание | Диапазон |
| :--- | :--- | :--- | :--- |
| `SMALLINT` | 2 байта | целое число малого диапазона | от -32,768 до +32,767 |
| `INTEGER` | 4 байта | стандартный выбор для целых чисел | от -2,147,483,648 до +2,147,483,647 |
| `BIGINT` | 8 байт | целое число большого диапазона | от -9,223,372,036,854,775,808 до +9,223,372,036,854,775,807 |
| `DECIMAL` | переменный | число с заданной точностью | до 131,072 цифр до десятичной точки; до 16,383 цифр после десятичной точки |
| `NUMERIC` | переменный | число с заданной точностью | до 131,072 цифр до десятичной точки; до 16,383 цифр после десятичной точки |
| `REAL` | 4 байта | число с переменной точностью | точность до 6 десятичных цифр |
| `DOUBLE PRECISION` | 8 байт | число с переменной точностью | точность до 15 десятичных цифр |
| `SMALLSERIAL` | 2 байта | малое автоинкрементное целое число | от 1 до 32,767 |
| `SERIAL` | 4 байта | автоинкрементное целое число | от 1 до 2,147,483,647 |
| `BIGSERIAL` | 8 байт | большое автоинкрементное целое число | от 1 до 9,223,372,036,854,775,807 |

### Булевые данные

В PostgreSQL представлен тип данных `BOOLEAN`, который может принимать значения TRUE, FALSE или NULL. Важным является факт, что `NULL` означает «неизвестно», что логически отличается от `FALSE`.

### Временные данные

Типы данных даты и времени в PostgreSQL и их описание:

| Имя | Размер | Описание | Минимальное значение | Максимальное значение |
| :--- | :--- | :--- | :--- | :--- |
| `TIMESTAMP [WITHOUT TIME ZONE]` | 8 байт | дата и время (без указания часового пояса) | 4713 год до н. э. | 294276 год н. э. |
| `TIMESTAMP WITH TIME ZONE` | 8 байт | дата и время с указанием часового пояса | 4713 год до н. э. | 294276 год н. э. |
| `DATE` | 4 байта | дата (без указания времени суток) | 4713 год до н. э. | 5,874,897 год н. э. |
| `TIME [WITHOUT TIME ZONE]` | 8 байт | время суток (без указания даты) | 00:00:00 | 24:00:00 |
| `TIME WITH TIME ZONE` | 12 байт | время суток (без даты), с указанием часового пояса | 00:00:00+1559 | 24:00:00-1559 |
| `INTERVAL` | 16 байт | временной интервал | -178,000,000 лет | +178,000,000 лет |

В большинстве случаев сочетание `DATE`, `TIME`, `TIMESTAMP WITHOUT TIME ZONE` и `TIMESTAMP WITH TIME ZONE` должно обеспечивать полный набор функций для работы с датой и временем, необходимых для любого приложения.


## Создание и заполнение базы данных

В рамках данного курса в качестве СУБД мы будем использовать **PostgreSQL**. Процесс установки мы опустим и сосредоточимся непосредственно на работе с данными. Для взаимодействия с базой вы можете выбрать любой удобный инструмент: графический клиент pgAdmin или консольную утилиту psql.

Для создания базы данных используется команда `CREATE DATABASE`. В качестве примера создадим базу данных с именем Students:
```
CREATE DATABASE "Students"
```

Для просмотра существующих баз данных с помощью psql необходимо выполнить команду `\l`.

Если же потребуется удалить базу данных, то можно воспользоваться оператором `DROP`. Для нашего примера это будет выглядеть следующим образом:
```
DROP DATABASE "Students";
```

Перед созданием первой таблицы необходимо подключиться к базе данных, в которую будет записана таблица. Для подключения к созданной базе данных используйте команду `\c`:
```
\c Students
```

### Создание таблиц

Сначала создадим родительскую таблицу для хранения информации об учебных группах. Воспользуемся оператором `CREATE`:
```
CREATE TABLE groups (
    group_id SERIAL PRIMARY KEY,
    group_name VARCHAR(50) NOT NULL UNIQUE,
    enrollment_year INTEGER NOT NULL
);
```

Разберём запрос:
- `CREATE TABLE groups` - создание таблицы с именем `groups`.
- `group_id SERIAL PRIMARY KEY` - создание столбца `group_id`, который имеет тип данных `SERIAL` (целое число с автоинкрементированием). `PRIMARY KEY` является дополнительным параметром и указывает, что столбец является первичным ключом.
- `group_name VARCHAR(50) NOT NULL UNIQUE` - столбец названия группы, который имеет строковый тип данных с переменной длиной до 50 символов. `NOT NULL` - дополнительный параметр, который запрещает пустые значения в столбце, `UNIQUE` - дополнительный параметр, который гарантирует уникальность значений в столбце.
- `enrollment_year INTEGER NOT NULL` - столбец для года зачисления (целочисленный тип данных, без пустых значений).

Для просмотра описания таблицы, можно воспользоваться командой `\d` с указанием таблицы, в нашем случае необходимо ввести команду `\d groups`.

Также создадим дочернюю таблицу для студентов, которая будет ссылаться на таблицу групп через внешний ключ:
```
CREATE TABLE students (
    student_id SERIAL PRIMARY KEY,
    first_name VARCHAR(20) NOT NULL,
    last_name VARCHAR(20) NOT NULL,
    group_id INTEGER REFERENCES groups(group_id) ON DELETE SET NULL
);
```

Разберём новые элементы:
- `REFERENCES` - ключевой слово, которое указывает, что числа должны соответствовать значениям в столбце `group_id` родительской таблицы `groups`.
- `ON DELETE SET NULL` - опция, которая устанавливает `NULL` для всех удалённых групп.

### Заполнение данными

Наполним наши таблицы данными с помощью оператора `INSERT`. Добавим группы в таблицу `groups`:
```
INSERT INTO groups (group_name, enrollment_year) 
VALUES ('УКФС-01-23', 2023),
('ПИ-33-22', 2022),
('ТИИ-02-21', 2021);
```
- После оператора `INSERT` указывается таблица и поля, в которые будут вставлены данные. Столбец `group_id` мы не указываем, так как для него работает автоинкрементирование.
- `VALUES` - ключевое слово, после которого в скобках перечисляются наборы значений для каждой новой строки, разделённые запятыми.

Просмотреть результат заполнения можно с помощью запроса:
```
SELECT *
FROM groups;
```

Добавим студентов в таблицу `students`:
```
INSERT INTO students (first_name, last_name, group_id) 
VALUES ('Иван', 'Петров', 1),
('Мария', 'Сидорова', 1),
('Алексей', 'Кузнецов', 2),
('Елена', 'Васильева', 3);
```

Давайте немного забежим вперёд и выведем полную информацию о студенте, для этого воспользуемся запросом:
```
SELECT *
FROM students AS s
LEFT JOIN groups AS g ON s.group_id = g.group_id;
```

Результатом данного запроса является вся информация о студентах, которую мы занесли в БД:
```
 student_id | first_name | last_name | group_id | group_id | group_name | enrollment_year
------------+------------+-----------+----------+----------+------------+-----------------
          1 | Иван       | Петров    |        1 |        1 | УКФС-01-23 |            2023
          2 | Мария      | Сидорова  |        1 |        1 | УКФС-01-23 |            2023
          3 | Алексей    | Кузнецов  |        2 |        2 | ПИ-33-22   |            2022
          4 | Елена      | Васильева |        3 |        3 | ТИИ-02-21  |            2021
(4 строки)
```
