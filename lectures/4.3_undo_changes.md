# Лекция 18 - Отмена изменений в Git

Система контроля версий Git предоставляет несколько механизмов для отмены изменений на разных этапах рабочего процесса.

## HEAD

HEAD - это символьная ссылка на ветку (или напрямую на коммит). Символьная ссылка отличается от обычной тем, что она содержит не сам хэш, а указатель на другую ссылку.

### HEAD как навигатор в истории Git

Для отмены изменений или создания новых веток часто нужно ссылаться не на последний коммит, а на его предков. Для этого необходимо использовать HEAD с символом `~`, например, `HEAD~1` указывает на коммит перед HEAD, а `HEAD~2` указывает на коммит, который находится на два коммита назад до HEAD.

## Reset

Команда `git reset` перемещает указатель HEAD на указанный коммит, по-разному обрабатывая изменения. У `git reset` есть три режима работы:
#### 1. `--soft` не затрагивает индекс и рабочую директорию, но переносит HEAD на указанный коммит. Рассмотрим пример:

```
$ git status
On branch main
Your branch is up to date with 'git_test/main'.

Changes to be committed:
  (use "git restore --staged <file>..." to unstage)
        new file:   new_data.csv

Untracked files:
  (use "git add <file>..." to include in what will be committed)
        new_data_1.csv

$ git log --oneline
28e11d4 (HEAD -> main, git_test/main, git_test/HEAD) Add files via upload
6ee228e Add files via upload
9d5b98a start point

$ git reset --soft 6ee228e

$ git status
On branch main
Your branch is behind 'git_test/main' by 1 commit, and can be fast-forwarded.
  (use "git pull" to update your local branch)

Changes to be committed:
  (use "git restore --staged <file>..." to unstage)
        new file:   new_data.csv
        new file:   some_data.json

Untracked files:
  (use "git add <file>..." to include in what will be committed)
        new_data_1.csv

$ git log --oneline
6ee228e (HEAD -> main) Add files via upload
9d5b98a start point

$ git reset --soft 28e11d4

$ git log --oneline
28e11d4 (HEAD -> main, git_test/main, git_test/HEAD) Add files via upload
6ee228e Add files via upload
9d5b98a start point
```

В данном примере у нас имеется 1 файл в индексе и 1 файл в рабочей области (не отслеживается), затем с помощью команды `git reset --soft` мы переходим на коммит назад (вместо хэша можно было бы написать `HEAD~1`), при этом из индекса и рабочей директории файлы не удалились. Просмотрев логи, мы видим, что коммит спереди пропал, однако мы все равно можем к нему перейти по хэшу.

#### 2. `--mixed` кроме перемещения HEAD сбрасывает индекс, но не затрагивает рабочую директорию, т. е. изменение в файлах сохраняется, но не помечается для коммита. Этот режим устанавливается по умолчанию (если не указывать режим, будет использован именно он). Рассмотрим пример:

```
$ git status
On branch main
Your branch is up to date with 'git_test/main'.

Changes to be committed:
  (use "git restore --staged <file>..." to unstage)
        new file:   new_data.csv

Untracked files:
  (use "git add <file>..." to include in what will be committed)
        new_data_1.csv

$ git log --oneline
28e11d4 (HEAD -> main, git_test/main, git_test/HEAD) Add files via upload
6ee228e Add files via upload
9d5b98a start point

$ git reset --mixed HEAD~1

$ git status
On branch main
Your branch is behind 'git_test/main' by 1 commit, and can be fast-forwarded.
  (use "git pull" to update your local branch)

Untracked files:
  (use "git add <file>..." to include in what will be committed)
        new_data.csv
        new_data_1.csv
        some_data.json

nothing added to commit but untracked files present (use "git add" to track)

$ git log --oneline
6ee228e (HEAD -> main) Add files via upload
9d5b98a start point
```

В отличие от прошлого примера, после `git reset` все файлы, отличные от коммита, стали неотслеживаемыми.

#### 2. `hard`, как можно было догадаться, сбрасывает индекс и рабочую директорию. Данный режим следует использовать с осторожностью, т. к. он удаляет изменения на уровне файловой системы. Рассмотрим пример:

```
$ git status
On branch main
Your branch is up to date with 'git_test/main'.

Changes to be committed:
  (use "git restore --staged <file>..." to unstage)
        new file:   new_data.csv

Untracked files:
  (use "git add <file>..." to include in what will be committed)
        new_data_1.csv

$ git log --oneline
28e11d4 (HEAD -> main, git_test/main, git_test/HEAD) Add files via upload
6ee228e Add files via upload
9d5b98a start point

$ git reset --hard HEAD~1
HEAD is now at 6ee228e Add files via upload

$ git status
On branch main
Your branch is behind 'git_test/main' by 1 commit, and can be fast-forwarded.
  (use "git pull" to update your local branch)

Untracked files:
  (use "git add <file>..." to include in what will be committed)
        new_data_1.csv

nothing added to commit but untracked files present (use "git add" to track)

$ ls
app.py  dockerfile  new_data_1.csv  requirements.txt  templates/
```

Как можно заметить из примера, `git reset --hard` удаляет только отслеживаемые файлы, но не трогает неотслеживаемые.

## Revert

Команда `git revert` создаёт новый коммит, который отменяет изменения, внесенные указанным коммитом, не удаляя его из истории. Рассмотрим пример:
```
$ git log --oneline
2a5628a (HEAD -> main) add new_data
28e11d4 (git_test/main, git_test/HEAD) Add files via upload
6ee228e Add files via upload
9d5b98a start point

$ git revert HEAD
[main 377e890] Revert "add new_data"
 1 file changed, 20641 deletions(-)
 delete mode 100644 new_data.csv

$ ls
app.py  dockerfile  requirements.txt  some_data.json  templates/

$ git log --oneline
377e890 (HEAD -> main) Revert "add new_data"
2a5628a add new_data
28e11d4 (git_test/main, git_test/HEAD) Add files via upload
6ee228e Add files via upload
9d5b98a start point

$ git reset --hard HEAD~1
HEAD is now at 2a5628a add new_data

$ ls
app.py  dockerfile  new_data.csv  requirements.txt  some_data.json  templates/
```

С помощью команды `git revert HEAD` мы отменили добавления файла `new_data.csv`, создав новый коммит, который является инверсией предыдущего коммита.
