# Лекция 12 - Межсетевой экран

Межсетевой экран - это программный или программно-аппаратный комплекс, который контролирует и фильтрует сетевой трафик, проходящий через него, в соответствии с заданными правилами.

iptables - это утилита командной строки для настройки и управления таблицами правил фильтрации пакетов в ядре Linux.

## Ключевые концепции iptables

### Правила

iptables оперирует некими правилами (rules), на основании которых принимается решение, что делать с поступившим пакетом. 

Правило состоит из:

- Условия (Match). Определяет критерии для отбора пакетов, например, условием может быть наличие пакета на порте 22.
- Действие (Target). Определяет операцию, которую нужно выполнить с пакетом, который подошёл по критериям отбора, например, отбросить пакет.
- Счётчика. Фиксирует количество срабатываний действия и суммарный объём пакетов в байтах.

### Цепочки

Набор правил формируется в цепочки. Пакет проходит через цепочку правил, и его судьба решается правилами по порядку. Базовые цепочки - набор правил, которые существуют в iptables по умолчанию. Существует 5 базовых цепочек правил:

- `INPUT`. Для пакетов, предназначенных этому хосту.
- `OUTPUT`. Для пакетов, которые генерирует этот хост.
- `FORWARD`. Для пакетов, которые проходят через этот хост.
- `PREROUTING`. Для пакетов, которые поступают на сетевой интерфейс.
- `POSTROUTING`. Для пакетов, которые покидают сетевой интерфейс.

### Таблицы

Таблица - набор базовых и пользовательских цепочек. Таблицы определяют общую цель правил.

Существует 5 таблиц правил:

- `filter` - таблица, выполняющая фильтрацию пакетов (разрешить/запретить). Содержит цепочки: `INPUT`, `OUTPUT`, `FORWARD`.
- `raw` используется для исключения пакетов из отслеживания соединений. Содержит цепочки: `INPUT` и `OUTPUT`.
- `nat` используется для преобразования сетевых адресов (NAT). Содержит цепочки: `PREROUTING`, `POSTROUTING`, `OUTPUT`.
- `mangle` предназначена для специализированного изменения заголовков пакетов, например, TTL. Содержит цепочки: `PREROUTING`, `POSTROUTING`, `OUTPUT`, `FORWARD`, `INPUT`.
- `security` используется для взаимодействия с внешними системами безопасности, например с SELinux и AppArmor.

### Путь пакета

1. Входящий пакет попадает в цепочку `PREROUTING` таблиц: `raw`, `magle` и `nat`.
2. Если адрес назначения пакета - локальный, то пакет идёт в цепочку `INPUT` таблиц: `mangle` и `filter`. После обработки локальной системой исходящий пакет проходит через цепочку `OUTPUT` таблиц: `raw`, `mangle`, `nat`, `filter`.
3. Если же адрес входящего пакета - посторонний, то пакет проходит цепучку `FORWARD` таблиц: `mangle`, `filter`.
4. Перед отправкой все исходящие пакеты (прошедшие цепочки `OUTPUT` или `FORWARD`) проходят цепочку `POSTROUTING` таблиц: `mangle` и `nat`.

## NAT

NAT - механизм преобразования сетевых адресов в заголовках IP-пакетов при их прохождении через межсетевой экран. В iptables реализация NAT сосредоточена в таблице `nat`.

### Типы NAT

#### 1. Source NAT (SNAT)

Изменяет IP-адрес источника (source) пакета. Применяется в цепочке `POSTROUTING`.

Пример SNAT:
```
iptables -t nat -A POSTROUTING -s 192.168.1.0/24 -j SNAT --to-source 95.165.15.10
```

Пакеты с адресом источника из сети `192.168.1.0/24` получают новый source-адрес `95.165.15.10`.

#### 2. Destination NAT (DNAT)

Изменяет адрес назначения (destination) пакета. Применяется в цепочке `PREROUTING` (для входящих) или `OUTPUT` (для исходящих).

Пример:
```
iptables -t nat -A PREROUTING -d 95.165.15.10 -p tcp --dport 80 -j DNAT --to-destination 192.168.1.100:80
```

Все входящие соединения на `95.165.15.10` перенаправить на `192.168.1.100:80`.

#### 3. Static NAT

Устанавливает постоянное однозначное соответствие между внутренним и внешним адресом. Каждому внутреннему хосту выделяется внешний публичный IP-адрес.

Пример:
```
iptables -t nat -A PREROUTING -d 95.165.15.20 -j DNAT --to-destination 192.168.1.50
```

Весь трафик, входящий на `95.165.15.20`, перенаправляется на `192.168.1.50`.

#### 4. Dynamic NAT

Сопоставляет внутренние IP-адреса с внешними из списка доступных публичных адресов. Когда внутренний хост выходит в сеть, ему на время сессии выделяется один свободный адрес.

Пример:
```
iptables -t nat -A POSTROUTING -s 192.168.1.0/24 -o eth0 -j SNAT --to-source 95.165.15.10-95.165.15.15
```

Когда хост из внутренней сети `192.168.1.0/24` инициирует исходящее соединение iptables автоматически выбирает свободный адрес из списка `95.165.15.10`-`95.165.15.15`.

#### 5. PAT (Port Address Translation)

Одна из форм Dynamic NAT, в которой используется один публичный адрес, но разные порты источника. Позволяет сотням устройств во внутренней сети выходить в интернет через один публичный IP. Это основа работы домашних роутеров.

Пример:
```
iptables -t nat -A POSTROUTING -s 192.168.1.0/24 -o eth0 -j MASQUERADE
```
Все устройства из внутренней сети `192.168.1.0/24` выходят в интернет через внешний интерфейс `eth0`, используя единственный публичный IP-адрес этого интерфейса. `-j MASQUERADE` динамически назначает порты источника для разных внутренних хостов. Вместо `MASQUERADE` можно использовать `SNAT` с указанием одного адреса.

## Базовые команды iptables

#### Просмотр текущих правил
Чтобы просмотреть текущие правила, используйте команду `iptables -L`. Это покажет все правила в таблице filter. Чтобы просмотреть правила в другой таблице, используйте флаг `-t`, например, `iptables -t nat -L`.

#### Добавление новых правил

Чтобы добавить новое правило, используйте команду `iptables -A`. Например, чтобы заблокировать весь входящий трафик от IP-адреса 192.168.0.100, вы можете использовать следующую команду:

`iptables -A INPUT -s 192.168.0.100 -j DROP` 

В этой команде `-A INPUT` добавляет новое правило в цепочку INPUT, `-s 192.168.0.100` определяет исходный IP-адрес, а `-j DROP` указывает, что все пакеты, соответствующие этому правилу, должны быть отброшены.

#### Удаление правил
Чтобы удалить правило, вам нужно знать его номер в цепочке. Вы можете узнать номера правил, используя команду `iptables -L --line-numbers`. Затем вы можете удалить правило, используя команду `iptables -D` и номер правила. Например, чтобы удалить первое правило в цепочке INPUT, вы можете использовать следующую команду:

`iptables -D INPUT 1` 

#### Изменение политики по умолчанию
Вы можете изменить политику по умолчанию для цепочки, используя команду `iptables -P`. Например, чтобы отклонить все входящие пакеты по умолчанию, вы можете использовать следующую команду:

`iptables -P INPUT DROP` 

#### Сохранение и восстановление правил
По умолчанию, правила iptables не сохраняются после перезагрузки системы. Чтобы сохранить текущие правила, вы можете использовать команду `iptables-save`. Это выведет текущие правила в формате, который можно использовать для восстановления с помощью команды `iptables-restore`.

`iptables-save > /path/to/iptables/rules` 

Затем, чтобы восстановить правила, вы можете использовать следующую команду:

`iptables-restore < /path/to/iptables/rules`

## Расширенное использование iptables

#### Логирование

Iptables предоставляет возможность логирования пакетов, что может быть полезно для отладки и мониторинга. Для включения логирования используйте цель `LOG`. Например, чтобы залогировать все отброшенные пакеты, можно использовать следующую команду:

`iptables -A INPUT -j LOG --log-prefix "Dropped Packet: " iptables -A INPUT -j DROP` 

Здесь `--log-prefix "Dropped Packet: "` добавляет указанный префикс к каждому лог-сообщению, что упрощает фильтрацию и анализ логов.

#### Создание пользовательских цепочек

Iptables позволяет создавать пользовательские цепочки для более гибкого управления правилами. Например, вы можете создать цепочку для обработки всего трафика от определенного IP-адреса:

`iptables -N MY_CHAIN iptables -A MY_CHAIN -s 192.168.0.100 -j DROP iptables -A INPUT -j MY_CHAIN` 

В этом примере команда `-N MY_CHAIN` создает новую цепочку с именем MY_CHAIN. Затем добавляется правило, которое отбрасывает все пакеты от 192.168.0.100. Наконец, цепочка MY_CHAIN добавляется в цепочку INPUT, что означает, что все входящие пакеты будут также проходить через цепочку MY_CHAIN.

#### Использование состояний пакетов

Iptables может отслеживать состояние пакетов, что позволяет создавать более сложные правила. Например, вы можете разрешить только входящие пакеты, которые являются частью уже установленного соединения:

`iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT` 

В этом примере `-m state` указывает на использование модуля состояний, а `--state ESTABLISHED,RELATED` указывает, что правило применяется только к пакетам, которые являются частью уже установленного или связанного соединения.

#### Использование iptables для создания VPN

Iptables также может быть использован для создания VPN (Virtual Private Network). Это может быть полезно для создания безопасного соединения между удаленными системами. Например, вы можете использовать iptables для настройки IPsec VPN, используя модуль "policy" для определения правил обработки трафика.

`iptables -A INPUT -m policy --dir in --pol ipsec -j ACCEPT iptables -A OUTPUT -m policy --dir out --pol ipsec -j ACCEPT` 

В этом примере `-m policy` указывает на использование модуля "policy", `--dir in` и `--dir out` указывают на направление трафика, а `--pol ipsec` указывает на использование IPsec политики.

#### Использование iptables для ограничения скорости

Iptables также может быть использован для ограничения скорости сетевого трафика, что может быть полезно для управления пропускной способностью. Для этого можно использовать модуль "limit". Например, вы можете ограничить количество новых SSH-соединений до 3 в минуту следующим образом:

`iptables -A INPUT -p tcp --dport 22 -m state --state NEW -m limit --limit 3/min -j ACCEPT` 

В этом примере `-m limit --limit 3/min` указывает на ограничение скорости, а остальная часть команды определяет, что она применяется к новым SSH-соединениям.

#### Использование iptables для редиректа трафика

Iptables может быть использован для редиректа трафика, что может быть полезно для перенаправления трафика на другой порт или сервер. Например, вы можете перенаправить весь входящий трафик на порт 80 на порт 8080 следующим образом:

`iptables -t nat -A PREROUTING -p tcp --dport 80 -j REDIRECT --to-port 8080` 

В этом примере `-t nat` указывает на использование таблицы NAT, `-A PREROUTING` добавляет правило в цепочку PREROUTING, `--dport 80` указывает на исходный порт, а `--to-port 8080` указывает на порт назначения.

Iptables - это мощный и гибкий инструмент, который может быть использован для решения множества задач управления сетью. Однако, как и любой мощный инструмент, он требует тщательного понимания и практики для эффективного использования.

## Практические примеры использования

### Запрет ICMP-протокола

Для отбрасывания всех входящих ICMP-пакетов необходимо добавить правило:
```
$ iptables -A INPUT -p icmp -j DROP
```

Для проверки проверим утилиту ping:
```
$ ping google.com
...
--- google.com ping statistics ---
7 packets transmitted, 0 received, 100% packet loss, time 6138ms
```

Соответственно, чтобы запретить все исходящие пакеты нужно заменить `INPUT` на `OUTPUT`.

### Запрет порта 22 для конкретного IP

```
$ iptables -A INPUT -s 192.168.1.100 -p tcp --dport 22 -j DROP
```

`-s` указывает IP-адрес, которому запрещён доступ, а `--dport` применяет правило к трафику, входящему на порт 22.
