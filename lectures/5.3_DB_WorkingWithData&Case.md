# Лекция 24 - Работа с данными и условная логика

Данные редко хранятся в базах данных в том формате, в котором они нужны в результирующем наборе. В этой лекции мы рассмотрим, как работать, преобразовывать, создавать данные, а также ознакомимся с ветвлением в SQL.

## Строковые данные

При работе с символьными данными обычно используются следующие типы данных: `CHAR`, `VARCHAR` и `TEXT`. Подробнее про эти типы данных мы уже говорили в первой лекции по базам данных.

### Манипулирование строковыми данными

Для добавления записи используется оператор `INSERT`, чтобы заполнить только определенные столбцы их можно указать после название таблицы:
```
INSERT INTO students (first_name, last_name)
VALUES ('Андрей', 'Сидоров');
```

> Стоит не забывать, что `CHAR` и `VARCHAR` могут иметь ограничение по количеству символов. Если длина строки превышает максимальный размер символьного столбца, то сервер PostgreSQL выдаст ошибку о превышении, однако другие серверы могут просто усечь значение строки до предела.

Для изменения записей в таблице существует оператор `UPDATE`:
```
UPDATE students
SET last_name = 'Иванов'
WHERE student_id = 20;
```

Для удаления записей существует два оператора: `DELETE` и `TRUNCATE`. `DELETE` удаляет записи построчно и используется вместе с `WHERE` для фильтрации данных, если `WHERE` отсутствует, то удаляются все записи таблицы. `TRUNCATE` удаляет все записи быстрее, чем сделал бы это `DELETE`, а также не вызывает срабатывание триггеров и сбрасывает счётчик идентификаторов до начального значения.

Пример удаления записи с `DELETE`:
```
DELETE FROM students
WHERE student_id = 20;
```

### Работа со строками

Одна из наиболее используемых строковых функций - `LENGTH()`, возвращающая длину строки:
```
SELECT first_name, LENGTH(first_name) AS len_name
FROM Students
LIMIT 5;

 first_name | len_name
------------+----------
 Дмитрий    |        7
 Анна       |        4
 Сергей     |        6
 Ольга      |        5
 Павел      |        5
(5 строк)
```

Иногда может потребоваться определить, где в строке располагается подстрока. Для этого можно воспользоваться функцией `POSITION()`, которая возвращает первый начальный индекс подстроки или 0 (если подстрока отсутствует):
```
SELECT first_name, POSITION('Дм' IN first_name) AS pos
FROM Students
LIMIT 5;

 first_name | pos
------------+-----
 Дмитрий    |   1
 Анна       |   0
 Сергей     |   0
 Ольга      |   0
 Павел      |   0
(5 строк)
```

> Обратите внимание, что порядковый номер символов в строке начинается с 1.

Для объединения строк можно использовать два подхода: оператор `||` и функцию `CONCAT()`. Оператор `||` вернёт `NULL`, если в операндах встретился `NULL`, `CONCAT` игнорирует в аргументах `NULL`:
```
SELECT 'Имя ' || 'и фамилия' AS result;

    result
---------------
 Имя и фамилия
(1 строка)

SELECT 'Имя ' || NULL AS result;

 result
--------

(1 строка)

SELECT CONCAT('Имя ', NULL, 'и фамилия') AS result;

    result
---------------
 Имя и фамилия
(1 строка)
```

Для замены вхождения подстроки в строке существует функция `REPLACE()`:
```
SELECT replace('abcdefabcdef', 'cd', 'XX') AS result;

    result
--------------
 abXXefabXXef
(1 строка)
```

Для извлечения подстроки из строки можно использовать `SUBSTRING()`, которая извлекает указанное число символов, начиная с заданной позиции:
```
SELECT last_name, SUBSTRING(last_name FROM 5 for 3) as sub
FROM students
LIMIT 5;

 last_name | sub
-----------+-----
 Соколов   | лов
 Морозова  | зов
 Волков    | ов
 Зайцева   | ева
 Козлов    | ов
(5 строк)
```

Для приведения текста к единому виду можно изменить регистр строки с использованием функций `LOWER()` и `UPPER()`, которые переводят строку в нижний и верхний регистр соответственно:
```
SELECT first_name, LOWER(first_name) AS low_name, UPPER(first_name) AS up_name
FROM students
LIMIT 5;

 first_name | low_name | up_name
------------+----------+---------
 Дмитрий    | дмитрий  | ДМИТРИЙ
 Анна       | анна     | АННА
 Сергей     | сергей   | СЕРГЕЙ
 Ольга      | ольга    | ОЛЬГА
 Павел      | павел    | ПАВЕЛ
(5 строк)
```

Для разбиения строки можно использовать функцию `SPLIT_PART()`, которая разбивает строку по разделителю и возвращает n-ое поле после разбиения:
```
SELECT split_part('Иван,Дмитрий,Анна', ',', 2)  AS result;

 result
---------
 Дмитрий
(1 строка)
```

Существует много других функций для работы со строками, для более подробной информации обращайтесь к справочному руководству.

## Численные данные

Работа с числами значительно проще, чем со строками, как минимум в них нет регулярных выражений. Добавление, изменение и удаление численных данных аналогично строковым и не будет рассмотрено.

### Математические вычисления

В PostgreSQL и других СУБД существует большое множество встроенных функций для вычисления определенных математических операций, таких как корень, степень и тригонометрические функции. В таблице приведены распространенные числовые функции:

| Функция | Описание |
| :--- | :--- |
| `power(x, y)` или `pow(x, y)` | x в степени y |
| `div(x, y)` | Целочисленное деление |
| `exp(x)` | Экспонента |
| `ln(x)` | Натуральный логарифм |
| `log(b, x)` | Логарифм x по основанию b | 
| `pi()` | Число π |
| `acos(x)` | Арккосинус  |
| `asin(x)` | Арксинус |
| `atan(x)` | Арктангенс |
| `cos(x)` | Косинус |
| `cot(x)` | Котангенс |
| `sin(x)` | Синус |
| `tan(x)` | Тангенс |

### Округление чисел

Для ограничения чисел с плавающей точностью предназначены четыре функции: `CEIL()`, `FLOOR()`, `ROUND()`, `TRUNC()`.

Функции `CEIL()` и `FLOOR()` предназначены для округления вверх и вниз, до ближайшего целого:
```
SELECT FLOOR(4.5)  AS res_fl, CEIL(4.5) AS res_ceil;

 res_fl | res_ceil
--------+----------
      4 |        5
(1 строка)
```

Функция `ROUND()` округляет в большую или меньшую сторону от середины промежутка между двумя целыми:
```
SELECT ROUND(4.499) AS res1, ROUND(4.5) AS res2,  ROUND(4.501) AS res3;

 res1 | res2 | res3
------+------+------
    4 |    5 |    5
(1 строка)
```

Также можно с помощью `ROUND()` округлить до n знаков после запятой:
```
SELECT ROUND(4.499, 3) AS res1, ROUND(4.5, 2) AS res2,  ROUND(4.501, 1) AS res3;

 res1  | res2 | res3
-------+------+------
 4.499 | 4.50 |  4.5
(1 строка)
```

Функция `TRUNC()` отбрасывает лишние разряды без округления:
```
SELECT TRUNC(4.499, 2) AS res1, TRUNC(4.5, 0) AS res2,  TRUNC(4.501, 1) AS res3;

 res1 | res2 | res3
------+------+------
 4.49 |    4 |  4.5
(1 строка)
```

`ROUND()` и `TRUNC()` вторым аргументом могут принимать отрицательные значения, тогда цифры слева от десятичной точки становятся равными 0:
```
SELECT ROUND(415.499, -1) AS res1, TRUNC(415.5, -2) AS res2,  TRUNC(415.501, -3) AS res3;

 res1 | res2 | res3
------+------+------
  420 |  400 |    0
(1 строка)
```

### Работа со знаком

Для работы со знаком основными функциями являются `ABS()` и `SIGN()`. `ABS()` возвращает модуль числа, `SIGN()` возвращает `-1`, `0` или `+1` в зависимости от знака числа:
```
SELECT SIGN(101) AS sign_res1, SIGN(0) AS sign_res2,  SIGN(-500) AS sign_res3;

 sign_res1 | sign_res2 | sign_res3
-----------+-----------+-----------
         1 |         0 |        -1
(1 строка)

SELECT ABS(101) AS abs_res1, ABS(0) AS abs_res2,  ABS(-500) AS abs_res3;

 abs_res1 | abs_res2 | abs_res3
----------+----------+----------
      101 |        0 |      500
(1 строка)
```

### Генерация случайных чисел

Функция `RANDOM()` возвращает число от 0 до 1 (равномерное распределение):
```
SELECT RANDOM() AS res;

         res
---------------------
 0.03441092080254382
(1 строка)
```

Для выхода из диапазона [0, 1] можно использовать базовые математические операторы (`*`, `-`, `+`) или задать аргументы для функции `RANDOM()`, которые определяют диапазон:
```
SELECT RANDOM(1.0, 10.0) AS res;

 res
-----
 1.9
(1 строка)
```

Для генерации чисел с нормальным распределением используйте функцию `RANDOM_NORMAL()`. Функция `SETSEED()` устанавливает начальное значение для последующих вызовов функций `RANDOM()` и `RANDOM_NORMAL()`.

## Временные данные

### Часовые пояса

Мир разделён на 24 часовых пояса. В качестве точки отсчёта используется UTC (Coordinated Universal Time), представляющее собой "мировое время" без привязки к часовым поясам. Все остальные часовые пояса описываются как смещение относительно UTC. Смещение показывает, насколько часов местное время отличается от всемирного.

Чтобы просмотреть часовой пояс в PostgreSQL, можно воспользоваться следующей командой:
```
SHOW timezone;

   TimeZone
---------------
 Europe/Moscow
(1 строка)
```

Установить часовой пояс можно с помощью оператора `SET`:
```
SET TIMEZONE TO '0:00';

SHOW timezone;

 TimeZone
----------
 0:00
(1 строка)
```

Однако таким образом можно установить временную зону только для текущей сессии. Другим способом (для постоянной настройки) является определение параметра TimeZone в конфигурационном файле.

### Строковое представление временных данных

Хотя получить временные данные можно с помощью встроенных команд, однако часто будет более удобно задать с помощью строкового представления. Для этого необходимо понимать компоненты этого представления:

| Компонент | Описание                 |
|:---------:|:-------------------------|
|   YYYY    | Год, включая столетие    |
|    MM     | Месяц                    |
|    DD     | День                     |
|    hh     | Час                      |
|   hhh     | Часы (прошедшие)         |
|    mm     | Минута                   |
|    ss     | Секунда                  |

Для задания даты и времени используются следующие форматы:

| Тип       |   Формат            |
|:---------:|:--------------------|
|   Date    | YYYY-MM-DD          |
| timestamp | YYYY-MM-DD hh:mm:ss |
|   Time    | hhh:mm:ss           |

> Во многих СУБД различают `timestamp` и `datetime`, однако в PostgreSQL вместо `datetime` используется `timestamp with time zone`.

Для преобразования строки в дату можно воспользоваться функцией `CAST()`:
```
SELECT CAST('2026-01-21 15:00:00' AS TIMESTAMP);

      timestamp
---------------------
 2026-01-21 15:00:00
(1 строка)
```

### Встроенные функции для создания временных данных

Если же нужно сгенерировать временные данные из строки, которая не соответствует формату, чтобы использовать `CAST()`, то можно воспользоваться функцией `TO_DATE()`, которая конвертирует строку в формат `DATE`, основываясь на заданном формате:
```
SELECT TO_DATE('21 Jan 2026', 'DD Mon YYYY');

  to_date
------------
 2026-01-21
(1 строка)
```

Аналогичная функция есть для конвертации в `TIMESTAMP`:
```
SELECT TO_TIMESTAMP('21 Jan 2026', 'DD Mon YYYY');

      to_timestamp
------------------------
 2026-01-21 00:00:00+03
(1 строка)
```

Можно сгенерировать текущие временные данные с помощью функций:
```
SELECT CURRENT_DATE, CURRENT_TIME, CURRENT_TIMESTAMP;

 current_date |    current_time    |       current_timestamp
--------------+--------------------+-------------------------------
 2026-02-10   | 14:28:02.387301+03 | 2026-02-10 14:28:02.387301+03
(1 строка)
```

В функциях `CURRENT_TIME` и `CURRENT_TIMESTAMP` можно указать точность, задав аргумент:
```
SELECT CURRENT_TIME, CURRENT_TIME(0), CURRENT_TIMESTAMP, CURRENT_TIMESTAMP(0);

    current_time    | current_time |       current_timestamp       |   current_timestamp
--------------------+--------------+-------------------------------+------------------------
 14:30:47.894249+03 | 14:30:48+03  | 2026-02-10 14:30:47.894249+03 | 2026-02-10 14:30:48+03
(1 строка)
```

### Преобразование временных данных

Функция `DATE_ADD()` позволяет добавить `INTERVAL` к `TIMESTAMP WITH TOME ZONE`:
```
SELECT DATE_ADD('2026-01-21 15:00:00+03', '1 day');

        date_add
------------------------
 2026-01-22 15:00:00+03
(1 строка)
```

Оператор `AT TIME ZONE` конвертирует `TIMESTAMP` или `TIMESTAMPTZ` (с часовым поясом) в `TIMESTAMPTZ`,  скорректированное по указанному часовому поясу:
```
SELECT '2026-01-21 15:00:00+03' AT TIME ZONE 'UTC' AS UTC,
	'2026-01-21 15:00:00+03' AT TIME ZONE 'AST' AS AST;

         utc         |         ast
---------------------+---------------------
 2026-01-21 12:00:00 | 2026-01-21 08:00:00
(1 строка)
```

> Сервер PostgreSQL может различать типы `TIMESTAMP` и `TIMESTAMPTZ` по наличию смещения в строковом представлении, в примере выше это `+03`.

Функция `EXTRACT` позволяет извлечь интересующий компонент из `TIMESTAMP` или `INTERVAL`:
```
SELECT EXTRACT(YEAR from TIMESTAMP '2026-01-21 15:00:00+03');

 extract
---------
    2026
(1 строка)
```
Для вычисления разницы между `TIMESTAMP` используется базовый арифметический оператор (`-`):
```
SELECT TIMESTAMP '2026-01-21 15:00:00+03' - TIMESTAMP '2024-01-21 15:00:00+03';

 ?column?
----------
 731 days
(1 строка)
```

> Обратите внимание, что при расчёте разницы учитывается весокосный год (2024).

Для расчёта возраста можно использовать функцию `AGE()`, которая возвращает разницу между аргументами, используя годы и месяцы, а не только дни:
```
SELECT AGE(TIMESTAMP '2026-01-21 15:00:00+03', TIMESTAMP '2024-01-21 15:00:00+03');

   age
---------
 2 years
(1 строка)
```

## Преобразование типов данных

Мы уже использовали функцию `CAST()` для преобразования строки во временные данные, однако её можно использовать и для преобразования и других типов данных:
```
SELECT CAST(10.1 AS INTEGER),
	CAST(10.1 AS VARCHAR);

 int4 | varchar
------+---------
   10 | 10.1
(1 строка)
```

## Условная логика

Условная логика представляет собой возможность выбирать одно из направлений выполнения программы. В качестве примера рассмотрим разметку студентов по успеваемости:
```
SELECT 
  student_id,
  first_name,
  last_name,
  average_grade,
  CASE 
    WHEN average_grade >= 4.5 THEN 'Отличник'
    WHEN average_grade >= 3.5 THEN 'Хорошист'
    ELSE 'Низкая успеваемость'
  END AS acad_perf
FROM Students
ORDER BY average_grade DESC;

 student_id | first_name | last_name | average_grade | acad_perf
------------+------------+-----------+---------------+-----------
          8 | Ольга      | Зайцева   |          5.00 | Отличник
         13 | Артем      | Михайлов  |          4.90 | Отличник
          2 | Мария      | Сидорова  |          4.90 | Отличник
          6 | Анна       | Морозова  |          4.80 | Отличник
         18 | Татьяна    | Ковалева  |          4.70 | Отличник
          4 | Елена      | Васильева |          4.60 | Отличник
         12 | Екатерина  | Попова    |          4.60 | Отличник
          5 | Дмитрий    | Соколов   |          4.50 | Отличник
         16 | Алина      | Лебедева  |          4.40 | Хорошист
...
```

В данном примере использовалось выражение `CASE` для реализации логического ветвления:
- Если средний балл >= 4.5, то студент считается отличником.
- Иначе, если средний балл >= 4.5, то студент считается хорошистом.
- Иначе считается, что у студента низкая успеваемость.

В примере выше использовалось `CASE` с различными **условиями**, такую вариацию называют поисковая форма. Кроме поисковой формы существует и простая форма, она сравнивает выражение с набором **значений**:
```
SELECT 
  student_id,
  first_name,
  last_name,
  average_grade,
  CASE ROUND(average_grade)
    WHEN 5 THEN 'Отличник'
    WHEN 4 THEN 'Хорошист'
    ELSE 'Низкая успеваемость'
  END AS acad_perf
FROM Students
ORDER BY average_grade DESC;

 student_id | first_name | last_name | average_grade | acad_perf
------------+------------+-----------+---------------+-----------
          8 | Ольга      | Зайцева   |          5.00 | Отличник
         13 | Артем      | Михайлов  |          4.90 | Отличник
          2 | Мария      | Сидорова  |          4.90 | Отличник
          6 | Анна       | Морозова  |          4.80 | Отличник
         18 | Татьяна    | Ковалева  |          4.70 | Отличник
          4 | Елена      | Васильева |          4.60 | Отличник
         12 | Екатерина  | Попова    |          4.60 | Отличник
          5 | Дмитрий    | Соколов   |          4.50 | Отличник
         16 | Алина      | Лебедева  |          4.40 | Хорошист
...
```
