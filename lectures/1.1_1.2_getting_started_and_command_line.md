### 1. Выбор дистрибутива
> — Для начала нужно выбрать дистрибутив. Обрати внимание на популярные дистрибутивы для начинающих (Ubuntu, Linux Mint, elementary OS). Подскажу, что  Ubuntu и Linux Mint - отличные варианты для первого знакомства. Что из них выбрать? Выбирай, как мороженое: оба вкусные, но первый — классика, второй — с ягодкой. Бери Mint, если хочешь меньше танцев с настройками. Ubuntu — если готов к небольшим экспериментам.  

### 2. Установка дистрибутива Linux:
> — А теперь особое внимание на экран! Сейчас будем ставить твой первый Linux:

### 🔧 Вариант 1: Установка на реальный компьютер (физическая машина)

1. **Качаем образ**  
   — Идём на [ubuntu.com](https://ubuntu.com) или [linuxmint.com](https://linuxmint.com)  
   — Ищем кнопку "Download" (обычно она оранжевая и кричащая)  

2. **Делаем загрузочную флешку**  
   — Берем USB-флешку (минимум 4 ГБ)  
   — Качаем [Rufus](https://rufus.ie) (для Windows) или [BalenaEtcher](https://www.balena.io/etcher/)  
   — *Важно! Все данные с флешки удалятся.  

3. **Загрузка в BIOS/UEFI**  
   — Перезагружаем комп  
   — Жмём F2/DEL/F12 (зависит от материнки)  
   — В Boot Menu выбираем нашу флешку  

4. **Сама установка**  
   — Выбираем язык (если вдруг не знаешь английский)  
   — Для Dual Boot: **ОСТОРОЖНО** с разделами!  
   — *Показывает потертый стикер:* "Не трогай разделы с надписью NTFS — это Windows!"  

### 🖥️ Вариант 2: Виртуальная машина (безопасный способ)

1. **Ставим VirtualBox**  
   — Качаем с [virtualbox.org](https://www.virtualbox.org)  
   — Установка как обычная программа (просто жмём "Далее")  

2. **Создаём виртуальную машину**  
   — Жмём "Создать"  
   — Имя: "MyFirstLinux"  
   — Тип: Linux  
   — Версия: Ubuntu (64-bit)  
   — Памяти: минимум 2048 MB  
   — Жёсткий диск: 25 ГБ (VDI, динамический)  

3. **Запускаем установку**  
   — В настройках > Носители выбираем наш ISO  
   — Запускаем — появится экран установки  
   — Можно тыкать везде "Далее" (это же песочница!)  
 
> — "Если всё сломал — просто удали виртуальную машину и создай заново!"

### 3. Настройка SSH
> — Подключиться к нашей виртуалке можно двумя способами — как через парадную дверь, так и через секретный туннель.

### 🖥️ Способ 1: Простое подключение (через окно VirtualBox)
— Просто открываешь VirtualBox  
— Дважды кликаешь на нашу виртуалку "MyFirstLinux"  
— Всё! Ты внутри, как пингвин в холодильнике  

*(Вот так выглядит схема твоего подключения: [Комп] → [VirtualBox] → [Linux])*

### 🔐 Способ 2: SSH — наш секретный туннель (для крутых ребят)

1. **Настраиваем сеть в VirtualBox:**
   - Открываем настройки виртуалки
   - Сеть → Режим "Мост" (Bridged Adapter)
   - *Это как провести телефонную линию в нашу комнату*

2. **Ставим SSH-сервер в Linux:**
```bash
sudo apt update
sudo apt install openssh-server -y
sudo systemctl enable ssh
sudo systemctl start ssh
```
- *Выполняй команды по очереди!*
3. **Подключение по SSH**

**Для линуксоидов и маководов:**
```bash
ssh username@ip_address
```
-  Где username — твой логин в виртуалке (тот, что придумывал при установке)
-  А ip_address — адрес машины (узнаёшь командой ip addr)


**Для виндузятников:**

-  Качаем PuTTY 
-  Запускаем и вводим:
```bash
Host: ip_address
Port: 22
Тип подключения: SSH
```
- Жмём "Open"

**Первый раз спросит про 'fingerprint' — это нормально, жми 'Да' (но не в реальной жизни!)**

## 1. Первое что тебе предстоит освоить: Научиться перемещаться по файловой системе.🧭

— Для начала разберись, где ты находишься:

```bash

cd ~              # Возвращаемся домой 
cd ..             # Перейти в родительский каталог
```

## 2. Исследуем систему 🔍: 
— Давай научимся как просматривать содержимое директорий и файлов различными способами. 

```bash
ls <опции> <путь> - Вывести список файлов и каталогов.
ls -l - Вывести подробную информацию (права, размер, дата изменения и т.д.).
ls -a - Показать скрытые файлы и каталоги (начинающиеся с точки).
ls -t - Сортировать по времени изменения (последние измененные первыми).
ls -R - Рекурсивно вывести содержимое подкаталогов.

cat <файл> - Вывести содержимое файла на экран.less <файл> - Просмотреть файл постранично.
head <файл> - Вывести первые несколько строк файла.
tail <файл> - Вывести последние несколько строк файла.
file <файл> - Определить тип файла.
du <опции> <путь> - Оценить использование дискового пространства.
du -h - Вывести размер в человекочитаемом формате.
du -s - Вывести только суммарный размер каталога.
```
— Список достаточно большой, не пугайся, мы крутые ребята, все выучим.

## 3. Работа с файлами и каталогами 📂:
—  Тебе важно научиться создавать, удалять, переименовывать, копировать и редактировать файлы и каталоги.
Разберемся с этой темой вместе. Это как уборка в нашей комнате — нужно знать, что куда класть и что можно выбрасывать.

 ### Основные операции:
**Создание🏗️:** 
```bash
mkdir <путь> - Создать новый каталог.
mkdir -p <путь> - Создать все необходимые родительские каталоги.
touch <файл> - Создать пустой файл или изменить время последнего доступа.
```
**Удаление 🗑️:** 
```bash
rmdir <путь> - Удалить пустой каталог.
rm <опции> <путь> - Удалить файлы или каталоги.
rm -i <файл> - Запрашивать подтверждение перед удалением.
rm -r <каталог> - Удалить каталог рекурсивно (вместе со всем содержимым!). Осторожно!
rm -f <файл> - Удалить файл принудительно (без запроса подтверждения). Осторожно!
find . -name "*.txt" -exec rm {} \; - Удалить все файлы с расширением .txt (очень опасно!).
```
— Перед **rm -r** представь, что это МОИ вещи удаляешь. Трижды подумай!

**Копирование и перемещение 🚚:** 
```bash
cp -r <каталог> <назначение> - Скопировать каталог рекурсивно.
mv <источник> <назначение> - Переместить или переименовать файлы или каталоги.
cp <опции> <источник><назначение> - Скопировать файлы или каталоги.
```

**Поиск 🔍:**
```bash
find <путь> <опции> <действие> - Найти файлы и каталоги, соответствующие заданным критериям.
find . -name "*.txt" - Найти все файлы с расширением .txt в текущем каталоге и его подкаталогах.
find . -type d - Найти все каталоги в текущем каталоге и его подкаталогах.
find . -size +1M - Найти все файлы размером больше 1 мегабайта.
find . -mtime -7 - Найти все файлы, измененные за последние 7 дней.
```
**Редактирование ✏️:**
```bash
Можешь выбрать любой удобный редактор.
nano <файл>
vim <файл>
emacs <файл> 
```

## 4. Работа с командами 🛠️
— Теперь научимся создавать свои команды. Научимся использовать документацию и утилиты командной строки.

### Создание псевдонимов 🏷️
```bash
alias <имя>="команда" - Создать псевдоним для команды.
```
— Например, **alias la="ls -la"** создаст псевдоним la для команды ls -la.

###  Функции оболочки
```bash
man <команда> - Открыть руководство (manual) для команды.
apropos <ключевое_слово> - Поиск команд, связанных с ключевым словом.
```
— man-страницы — это как инструкция к моему старому тостеру: скучно, но спасает жизнь!

### Разбираемся с командами
``` bash
type cd    # Проверить тип команды
which ls   # Где живёт программа
```
— Знать разницу — всё равно что отличать мои носки от твоих. Важно для мирного сосуществования!

## 5.Перенаправление потоков 🌊
— Посмотрим базовые операции:
``` bash
ls > files.txt      # Сохранить вывод в файл
ls >> files.txt     # Добавить в конец файла
grep "текст" < file # Читать из файла
```
— Это как разливать мой бульон по контейнерам — главное не перепутай где что!"

### Работа с ошибками
``` bash
command 2> errors.log  # Сохранить ошибки
command &> all.log    # Всё в один файл
```

### Продвинутые приёмы
``` bash
ls | grep ".txt"    # Конвейер
cmd1 || cmd2       # Если первая команда провалилась
cmd1 && cmd2       # Если первая успешна
```
— Не переживай, если сразу не поймёшь. Я месяц путал > и >>, пока не пролил бульон на клавиатуру и не запомнил навсегда. 🐧✨

## 6. Горячие клавиши ⌨️

### Основные комбинации:
``` bash
Ctrl+C - Прервать выполняющуюся команду.
Ctrl+D - Завершить ввод (эквивалентно нажатию Enter на пустой строке).
Ctrl+Z - Остановить (suspend) выполняющуюся команду (можно продолжить выполнение с помощью fg или bg).
Ctrl+A - Переместить курсор в начало строки.
Ctrl+E - Переместить курсор в конец строки.
Ctrl+K - Удалить текст от курсора до конца строки.
Ctrl+U - Удалить текст от курсора до начала строки.
Ctrl+R - Поиск в истории команд.
Tab - Автодополнение имен файлов и команд.
Стрелки вверх/вниз - Переключение по истории команд.
``` 
— Эти комбинации спасли меня чаще, чем мой запасной свитер!

## 7. Ссылки 🔗
— Нам с тобой предстоит вместе понять разницу между символическими и жёсткими ссылками и научиться их создавать.

``` bash
ln <источник> <ссылка> - Создать жёсткую ссылку.
ln -s <источник> <ссылка> - Создать символическую ссылку (симлинк).
``` 
### Различия:
Жёсткая ссылка: 
- Создаёт новую запись в файловой системе, указывающую на те же данные, что и исходный файл. Нельзя создать жёсткую ссылку на каталог или на файл на другом разделе диска. При удалении исходного файла, жёсткая ссылка остаётся работоспособной.

Символическая ссылка: 
- Создаёт файл, содержащий путь к исходному файлу. Можно создавать на каталоги и файлы на других разделах. При удалении исходного файла, символическая ссылка становится "битой" (неработоспособной).

## 8. Планировщик Cron ⏰
— Cron - это планировщик задач, позволяющий автоматически запускать команды и скрипты в заданное время или с заданным интервалом.

``` bash
crontab -e - Редактировать crontab (файл, содержащий расписание задач).
crontab -l - Показать содержимое crontab.
crontab -r - Удалить crontab.
```
- Формат записи в crontab:
минута час день_месяца месяц день_недели команда

### Настройка заданий
``` bash
crontab -e  # Редактирование
0 * * * * command # Каждый час
```
