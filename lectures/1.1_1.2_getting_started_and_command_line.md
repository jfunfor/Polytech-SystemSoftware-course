# Лекция 1 + 2 - В добрый путь!

## Что такое Linux
Для начала разберемся, что такое Linux. Сейчас под этим словом понимают либо операционную систему, либо ядро. 

О ядре скажем две вещи. Это монолит, то есть оно включает в себя файловую систему, драйвера, [VFS (со ссылкой для любознательных)](https://habr.com/ru/companies/otus/articles/446614/), помимо обязательных и стандартных системы управления процессами, памятью, планировщика и диспетчера.

На этом курсе мы будем изучать не столько сам Linux, как ядро, сколько те программы, которые идут с ним в комплекте. Эти программы называются GNU Core Utilities и включают в себя все, что нужно системному администратору и рядовому пользователю. Обычно взаимодействие пользователя с этими программами происходит при помощи командной строки (эмулятора терминала). Открыв терминал, вы можете вписать туда команду и запустить ее.

## Bash

Используя терминал, мы неизбежно взаимодействуем с каким-либо интерпретатором командной строки. По умолчанию это чаще всего `bash`.

В лекциях мы часто будем взимодействовать с bash. Вы можете использовать интерпретатор в среде Windows поначалу. Например, такой идет вместе с дистрибутивом Git. Просто откройте меню "пуск" и запустите Git Bash. Конечно, иногда этого не будет хватать. Поэтому автор лекций советует держать у себя в запасе виртуальную машину с установленной Ubuntu 24 LTS. В качестве гипервизора рекомендуется использовать VMWare. Нам подойдет бесплатная версия "Player"

Давайте запустим терминал и проверим, какой же, все-таки у нас работает интерпретатор командной строки!

В общем случае "тело" команды выглядит так:

```
[Приложение] <флаги> <аргументы>
               ^       ^
               |       |
              --------------
              Не обязательно
              присутствуют
```

Напишем в терминал команду `ps` (текстовый аналог диспетчера задач) и нажмем клавишу Enter. Получим вывод, похожий на 

```
    PID TTY          TIME CMD
  26453 pts/0    00:00:00 bash
  26641 pts/0    00:00:00 ps
```

За исключением PID (ID процесса), все должно совпасть! Стоит отметить, что мы увидели не все процессы системы, а лишь те, что запущены в текущей среде.

При помощи командной строки можно выполнять любую задачу, которая выполнима через графический интерфейс. Со временем вы поймете, что многие задачи куда эффективнее выполнять именно последним способом.

Для начала изучим навигацию по директориям (то есть "папкам", но, прошу, не называйте их так). используем команду `cd` для перехода. Сначала определим, куда мы вообще можем перейти: `ls`

*Примечание: если в одном блоке ввода/вывода терминала будет и ввод, и вывод, то перед запускаемой командой будет ставиться значок `$`. Это означает, что запуск производится от лица обычного пользователя. `#` перед командой обозначает запуск от лица администратора (суперпользователя). Любая строчка без этих знаков - вывод приложения, которое было запущено.*

```sh
$ ls -l

drwxr-xr-x. 1 german german     86 Sep 11 18:42 Applications
drwxr-xr-x. 1 german german     72 Sep 14 21:41 Desktop
drwxr-xr-x. 1 german german     22 Sep 13 01:07 Documents
drwxr-xr-x. 1 german german   3992 Sep 19 19:37 Downloads
```

Разберем, что мы запустили и что получили. Команда `ls` выводит содержимое текущей директории среды. Также в поле ввода был добавлен флаг `-l`, который немного изменяет поведение команды, выводя дополнительную информацию списком. У каждой команды свой набор флагов. Попробуйте запустить команду в чистом виде.

В списке вывода найдите директорию. Можно легко определить ее по весу файла и цвету вывода. Директории обычно **синие**.

> Помимо имен файлов, в выводе так же можно наблюдать следующее: `drwxr...` - биты привелегий, далее владельца файла и группу-владельца. Затем вес и, наконец, дату изменения файла.

Перейдем в другую директорию. Для этого задействуем команду `cd` (change directory) и удостоверимся в правильности выполнения, выполнив `pwd`:

```
$ cd Downloads
$ pwd
/home/german/Downloads
```

Порядок! Теперь сделаем важную вещь. Снова задействуем команду `ls`, но в этот раз с флагами `-l -a`. Заметили изменения? В выводе появились директории `.` и `..`. Так обозначаются текущая директория и директория уровнем выше. Задействуем `cd`, чтобы **вернуться** назад в домашнюю директорию пользователя.

```
cd ..
```

В качестве альтернативы можно было бы указать **абсолютный** путь и точно так же перейти: `cd /home/<username>/`.

### Просмотр текстовых файлов в терминале

Существует много способов реализовать задуманное: есть множество утилит, которые мы планомерно разберем.

Наверное, самая базовая программа для просмотра текстовых файлов - `cat`. На вход в качестве аргумента поступает файл, которые требуется вывести. Например, для вывода в терминал системных логов:

```
$ sudo cat /var/log/syslog
```

Или для ОС на базе Red Hat:

```
$ sudo cat /var/log/messages 
```

Как вы заметили, в команду добавилось `sudo`. Эта утилита отвечает за выполнение команды от лица другого пользователя. По умолчанию, без аргументов sudo выполнит команду от лица пользователя `root`.

Как вы можете заметить, файл выводится полностью и это может быть не очень удобно в больших файлах

Поэтому есть более интерактивые способы, например, программа `less`. Ее запуск в базовом виде ничем не отличается от `cat`, но функционально она работает совсем иначе. Вы увидите начало файла и при помощи стрелочек на клавиатуре сможете осуществлять просмотр файла. Выйти из просмотра можно долистав до конца, либо нажав клавишу `q`.

И последний, но не по значимости, способ. Используем интерактивный редактор `vim` для прсмотра файлов. Откроем файл аналогично двум предыдущим программам. Используйте стрелочки для навигации по файлу. А вот как осуществить выход из `vim` вам придется разобраться самостоятельно :)

### Исследование файловой системы

Большинство дистрибутивов Linux, с точки зрения хранения файлов реализованы одинаково. Оно и неудивительно, учитывая логику данного ядра: все - файл (и процесс, и сетевой интерфейс). Разберем основные каталоги:

- `/` - корень ФС, точка ее монтирования
- `/home` - домашняя директория, содержащая пользовательские каталоги. Удобно, когда надо "выписать" старого пользователя из системы со всеми его файлами :)
- `/bin` - от англ. binary. Тут ОС хранит большинство бинарных файлов, начиная с некоторых системных программ, заканчивая пользовательскими приложениями.
- `/sbin` - большинство системных программ лежат по этому пути.
- `/opt` - от англ. optional. Служит хранилищем для программ, установленных беспорядочным образом (что есть порядочный - разберем в следующих лекциях)
- `/usr` - Unix System Resouces. Служит для хранения большинства пользовательских приложений и библиотек.
- `/lib` - Служит хранилищем библиотек для системного ПО
- `/boot` - Содержит файлы ядра и некоторые другие, необходимые для запуска ОС
- `/dev` - Хранилище файлов устройств. Разделяют special files на символьные и блочные. Второе устройство (block device) — то, которое выполняет ввод/вывод информационных данных в виде блоков (к примеру, SSD диск), а символьное устройство (symbol device) выполняет ввод/вывод информации в виде символов (например, клавиатура).
- `/proc` - это не что иное, как процесс. В этом каталоге содержатся псевдофайлы (pseudofiles), содержащие сведения о процессах и системных ресурсах.
- `/run` - В директорию монтируется виртуальная файловая система tmpfs для хранения файлов, связанных с выполняющимися процессами. Здесь хранятся сведения о времени выполнения оперативной памяти. Эти файлы хранятся в оперативной памяти и по завершению сеанса исчезают.
- `/root` - Это каталог для super user (администратор) — домашняя директория пользователя root.

## Операции с файлами и каталогами

Помимо редактирования файлов, их можно создавать, копировать, перемещать и удалять. 

Начнем с создания, которое происходит при помощи команды `touch`. Например, `touch grass.txt`. Эта команда создаст пустой файл с именем `grass.txt`. Создать директорию можно командой `mkdir`. Аргумент у нее аналогичен `touch`. Можно указать как относительный путь, так и абсолютный. При этом, если на каком-то участке пути директория не создана, то результатом исполнения будет ошибка. Чтобы избежать такой ситуации, надо использовать флаг `-p`.

Разберем создание директории по пути `/home/user/study/labs/lab1/code/`

```sh
$ mkdir /home/user/study/labs/lab1/code/
mkdir: cannot create directory ‘/home/user/study/labs/lab1/code/’: No such file or directory

$ mkdir -p /home/user/study/labs/lab1/code/
# Команда выполнена успешно
```

Разберем копирование. Для копирования файлов и директорий используется команда `cp`:

```sh
cp <путь_1> <путь_2>
```

Директория копируется немного иначе. Необходимо задействовать рекурсивное копирование:

```sh
cp -r <путь_1> <путь_2>
```

Также можно скопировать все поддиректории и файлы, содержащиеся в какой-либо директории:

```
cp -r ./dir1/logs/* ./dir2/
```

Тут все файлы и директории из `./dir/logs/` будут скопированы в `./dir2`. Уточним, что `./` - это текущая директория (помните `.` и `..`?), а `*` - это символ, указывающий на все файлы в директории. Если вместо `*` мы напишем `*.txt`, то выбраны и, соответственно скопированы, будут все файлы с расширением txt.

Перемещение выполняется командой `mv` при этом перемещать директории нужно без указания каких-либо дополнительных флагов. `mv` так же можно использовать для переименования.

Удаление происходит команой `rm`. В качестве аргумента точно так же указывается путь до удаляемого файла. Пустую директорию можно удалить командой `rmdir`. А если нужно удалить директорию со всем содержимым, то задействуйте флаг `-r` (тоже говорит о рекурсивном удалении). Можно так же указать флаг `-f`, чтобы заглушить предупреждения об удалении файлов. Будьте аккуратны с этим флагом, *с большой силой приходит большая ответственность*!

## Ссылки
Ссылки в Linux — это мощный инструмент, позволяющий указывать путь к файлам и каталогам. Существует два основных типа ссылок: жёсткие ссылки и мягкие ссылки (вторые также известны как символические ссылки или симлинки). Понимание различий между этими двумя типами поможет вам эффективно управлять файловой системой.

Каждый файл в ФС имеет свой индексный дескриптор (или inode): это специальная структура данных, которая хранит информацию (метаданные) о файле. Каждый индексный дескриптор имеет уникальный номер, по которому файловая система его и находит. 

Ссылка — это просто ещё одна запись в каталоге для файла или директории. Она находится между именем файла и фактическими данными, хранящимися в файловой системе. Создавая ссылки, мы можем иметь несколько записей в каталоге (имён) для одного и того же файла или директории, что позволяет использовать два или более имени для одного и того же объекта.

Жёсткая ссылка — это запись в каталоге, которая указывает непосредственно на инод файла. Она действует как отдельный файл и ссылается на конкретное место на жёстком диске. Это зеркальная копия оригинального файла. Давайте рассмотрим, как создавать жёсткие ссылки и узнаем, как они работают. Для создания жёсткой ссылки используется команда ln.

Жесткая ссылка создается командой `ln <original file> <hardlink>`

Мягкая ссылка (символическая ссылка) — это отдельный файл, содержащий путь к другому файлу или директории. В отличие от жёстких ссылок, мягкие ссылки не ссылаются напрямую на индексный дескриптор целевого файла, а имеют собственные индексные дескрипторы и содержат путь к целевому файлу или каталогу.

Чтобы создать мягкую ссылку, задействуйте `ln -s <original> <symlink>`

Подробнее о ссылках вы можете почитать в сети Интернет или документации `ln` - `$ man ln`

## Перенаправление

Настало время достать скелета из шкафа и признаться: все это время мы работали с потоками stdin, stdout, stderr, которые представляют собой три стандартных потока данных (стандартный ввод, вывод, вывод ошибок соответственно). По умолчанию stdin читает информацию с клавиатуры, два других выводят информацию на экран. Такой функционал был бы избыточен, если бы не функция перенаправления потока. `echo`, кстати, специализируется на выводе текста в stdout.

```
echo "Опухший мужик есть Киндер Буэно, на пляже в Сочи вода по колено" > out.txt
```

Оператор перенаправления `>` без аргументов направит stdout в файл. Этот оператор перезапишет содержимое файла на stdout. Точно так же можно перенаправить и другой поток, например `2>` перенаправит stdout. А оператор `>>` добавит входной stdout в конец файла вместо перезаписи.

Оператор `|` (pipe) перенаправляет stdout команды справа в stdin команды слева. Тем самым можно осуществлять простую фильтрацию командой `grep`. Выведем все строки, содержащие сообщения об ошибках из системного лога.

```
# cat /var/log/syslog | grep error
```

## Особенности работы с клавиатурой

Также представлю немного полезных сочетаний клавиш напоследок:
- `Ctrl + L` - Очистка терминала
- При помощи стрелочек можно смотреть историю команд
- Используйте Tab для автозаполнения путей, имен команд и не только!
- `Ctrl + W` - удалить целое слово за курсором
- `Ctrl + D` - удалить целое слово перед курсором
