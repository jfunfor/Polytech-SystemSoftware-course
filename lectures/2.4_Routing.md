# Лекция 11 - Маршрутизация в Linux

Маршрутизация - процесс определения пути, по которому будут отправляться сетевые пакеты. 

## Таблица маршрутизации

Таблица маршрутизации - электронная таблица, в которой описываются соответствия между адресами назначения и интерфейсами, через которые отправляются пакеты данных к следующему маршрутизатору.

Для вывода таблицы можно воспользоваться командой `ip route`:
```
$ ip route

default via 10.0.2.2 dev enp0s3 proto dhcp src 10.0.2.15 metric 100
10.0.2.0/24 dev enp0s3 proto kernel scope link src 10.0.2.15 metric 100
```

Первая запись в таблице маршрутизации является маршрутом по умолчанию, т. е. все пакеты не попадающие под другие правила, будут направляться по данному правилу через шлюз `10.0.2.2` с интерфейса `enp0s3`. Вторая запись предназначена для подсети `10.0.2.0/24` и указывает, что пакеты будут передаваться напрямую через интерфейс `enp0s3` без использования шлюза.

## Добавление и удаление маршрутов

### Статические маршруты

Статический маршрут - это маршрут, прописанный администратором вручную.

Для добавления маршрута используется команда:
```
$ ip route add <сеть-назначения> via <адрес-шлюза>
```
Необязательными параметрами являются интерфейс (dev), метрика (metric) и протокол (proto). Таким образом, синтаксис команды принимает вид:
```
$ ip route add <сеть-назначения> via <адрес-шлюза> [dev <интерфейс>] [metric <метрика>] [proto <протокол>]
```

Пример использования:
```
$ ip route add 192.168.0.1 via 10.0.2.2
$ ip route

default via 10.0.2.2 dev enp0s3 proto dhcp src 10.0.2.15 metric 100
10.0.2.0/24 dev enp0s3 proto kernel scope link src 10.0.2.15 metric 100
192.168.0.1 via 10.0.2.2 dev enp0s3
```

Для удаления маршрута `add` заменяется на `del`:
```
$ ip route del 192.168.0.1 via 10.0.2.2
$ ip route

default via 10.0.2.2 dev enp0s3 proto dhcp src 10.0.2.15 metric 100
10.0.2.0/24 dev enp0s3 proto kernel scope link src 10.0.2.15 metric 100
```

### Маршруты по умолчанию

Маршрут по умолчанию - это маршрут, который используется, когда нет других подходящих маршрутов.

Удаление маршрута по умолчанию:
```
$ ip route del default via 10.0.2.2
$ ip route

10.0.2.0/24 dev enp0s3 proto kernel scope link src 10.0.2.15 metric 100
```

Добавление маршрута по умолчанию:
```
$ ip route add default via 10.0.2.1
$ ip route

default via 10.0.2.1 dev enp0s3
default via 10.0.2.2 dev enp0s3 proto dhcp src 10.0.2.15 metric 100
10.0.2.0/24 dev enp0s3 proto kernel scope link src 10.0.2.15 metric 100
```

Для определения нескольких маршрутов по умолчанию необходимо назначать метрику в таблице, которая определяет приоритет (чем больше значение метрики, тем ниже приоритет). Так как для первой записи метрика не указана явно, то она принимает значение 0, следовательно, первая запись является более приоритетной, чем вторая.

Чтобы изменить существующую запись, можно использовать команду `ip route replace`:
```
$ ip route replace default via 10.0.2.3
$ ip route

default via 10.0.2.3 dev enp0s3
default via 10.0.2.2 dev enp0s3 proto dhcp src 10.0.2.15 metric 100
10.0.2.0/24 dev enp0s3 proto kernel scope link src 10.0.2.15 metric 100
```

## Форвардинг пакетов

Форвардинг пакетов - функция, позволяющая пересылать пакеты между сетевыми интерфейсами, превращая систему в маршрутизатор.

Для проверки текущего статуса форвардинга можно воспользоваться командой:
```
$ cat /proc/sys/net/ipv4/ip_forward

0
```

В выводе `0` означает то, что форвардинг выключен. Для включения форвардинга воспользуемся следующей командой:
```
$ echo 1 | sudo tee /proc/sys/net/ipv4/ip_forward
```

Однако такой способ не сохраняется на постоянной основе, при перезагрузке настройка сбросится:
```
$ sudo reboot
$ cat /proc/sys/net/ipv4/ip_forward

0
```

Для включения форвардинга на постоянной основе необходимо отредактировать файл `/etc/sysctl.conf` и раскомментировать строку `net.ipv4.ip_forward=1`, затем применить изменения:

```
$ sudo nano /etc/sysctl.conf            # Редактирование файла
$ sudo sysctl -p /etc/sysctl.conf       # Применение изменений
$ cat /proc/sys/net/ipv4/ip_forward     # Проверка

1

$ sudo reboot
$ cat /proc/sys/net/ipv4/ip_forward

1
```
