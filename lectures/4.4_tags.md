# Лекция 19 - Тэги в Git

Тэги - это указатели на определенные коммиты в истории репозитория, которые используются для пометки важных моментов в разработке, например, релиз версии. Тэг похож на ветку, которая никогда не перемещается: он всегда указывает на один и тот же коммит, давая ему понятное имя.

## Классификация тэгов

Тэги бывают двух типов: аннотированные и легковесные. **Легковесные** тэги просто указывают на конкретный коммит и не содержат дополнительной информации. Тэг можно создать с помощью команды `git tag`:
```
$ git log --oneline
2a5628a (HEAD -> main) add new_data
28e11d4 (git_test/main, git_test/HEAD) Add files via upload
6ee228e Add files via upload
9d5b98a start point

$ git tag v1.0 9d5b98a

$ git log --oneline
2a5628a (HEAD -> main) add new_data
28e11d4 (git_test/main, git_test/HEAD) Add files via upload
6ee228e Add files via upload
9d5b98a (tag: v1.0) start point

$ git checkout v1.0
Previous HEAD position was 2a5628a add new_data
HEAD is now at 9d5b98a start point
```

В этом примере мы добавляем простой легковесный тэг (tag: v1.0) коммиту с хэшем `9d5b98a`, а затем используем для переключения на коммит по тэгу.

**Аннотированные** тэги имеют более сложную структуру. При создании аннотированного тэга Git создаёт полноценный объект и указывающую на него ссылку, а не просто указатель на коммит. Для создания аннотированного тэга необходимо добавить к команде опцию `-a`:
```
$ git tag -a v1.1 -m "Version 1.1"

$ git tag
v1.0
v1.1

$ git log --oneline
2a5628a (HEAD, tag: v1.1, main) add new_data
28e11d4 (git_test/main, git_test/HEAD) Add files via upload
6ee228e Add files via upload
9d5b98a (tag: v1.0) start point
```

В этом примере мы не указывали ссылку на коммит, поэтому по умолчанию использовался HEAD, также мы сразу же указали сообщение для аннотированного тэга, аналогично тому, как мы делали для веток. Кроме сообщения аннотированные тэги содержат информацию об авторе и дате. Команда `git tag` также выводит список всех тэгов.

Для удаления локального тэга к команде `git tag` необходимо добваить опцию `-d` или `--delete`:
```
$ git tag -d v1.1
Deleted tag 'v1.1' (was e0c9945)

$ git log --oneline
2a5628a (HEAD, main) add new_data
28e11d4 (git_test/main, git_test/HEAD) Add files via upload
6ee228e Add files via upload
9d5b98a (tag: v1.0) start point
```

## Работа с тэгами в удаленном репозитории

Для отправки тэга в удаленный репозиторий необходимо воспользоваться командой `git push` с указанием удаленного репозитория и тэга:
```
$ git push git_test v1.0
```
Вместо конкретного тэга можно указать опцию `--tags` для отправки всех тэгов.

Для получения тэгов с удаленного репозитория можно воспользоваться командой `git fetch`:
```
$ git tag -d v1.0
Deleted tag 'v1.0' (was 9d5b98a)

$ git log --oneline
2a5628a (HEAD -> main, git_test/main, git_test/HEAD) add new_data
28e11d4 Add files via upload
6ee228e Add files via upload
9d5b98a start point

$ git fetch git_test --tags
From https://github.com/<git_username>/git_test
 * [new tag]         v1.0       -> v1.0

$ git log --oneline
2a5628a (HEAD -> main, git_test/main, git_test/HEAD) add new_data
28e11d4 Add files via upload
6ee228e Add files via upload
9d5b98a (tag: v1.0) start point
```

Для удаления тэга с удаленного репозитория необходимо воспользоваться командой `git push` с опцией `-d`:
```
$ git push git_test -d v1.0
To https://github.com/<git_username>/git_test
 - [deleted]         v1.0

$ git tag -d v1.0
Deleted tag 'v1.0' (was 9d5b98a)

$ git log --oneline
2a5628a (HEAD -> main, git_test/main, git_test/HEAD) add new_data
28e11d4 Add files via upload
6ee228e Add files via upload
9d5b98a start point

$ git fetch git_test --tags

$ git log --oneline
2a5628a (HEAD -> main, git_test/main, git_test/HEAD) add new_data
28e11d4 Add files via upload
6ee228e Add files via upload
9d5b98a start point
```
