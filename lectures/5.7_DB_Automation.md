# Лекция 28 - Автоматизация в SQL

## Процедуры и функции

Часто бывает потребность в выполнении определенной последовательности действий с данными: добавить нового студента, записать на курс или вычислить средний балл. Вместо того чтобы много раз писать запросы, можно сохранить алгоритм действий в сервере БД.

Хранимая процедура - это набор SQL-инструкций, который хранится и выполняется на сервере БД. Она может принимать параметры, изменять данные, но, в отличие от функции, не возвращает значение.

Функция - это также именованный блок кода, который обязательно возвращает значение.

Для написания процедур и функций в PostgreSQL обычно используется язык PL/pgSQL - это процедурный язык, который добавляет к обычному SQL условные операторы, обработку ошибок и другие элементы программирования недоступные в SQL.

### Создание процедуры

Для создания процедуры используется команда `CREATE PROCEDURE`:
```
CREATE PROCEDURE name(args)
LANGUAGE plpgsql
AS $$
DECLARE
    -- Объявление переменных
BEGIN
    -- Тело процедуры
END;
$$;
```

Основные элементы:
- `CREATE PROCEDURE name (args)` - определяет имя процедуры и аргументы.
- `LANGUAGE plpgsql` - указывает язык, на котором написано тело функции. Кроме PL/pgSQL, доступны: PL/Tcl, PL/Perl и PL/Python.
- `AS $$ ... $$;` - определяет границы тела функции. Вместо `$$` можно использовать и другие символы, например `'`, но кавычки неудобны тем, что они мешают использованию строк внутри процедуры.
- `DECLARE` - блок для объявления локальных переменных.
- `BEGIN ... END;` - блок, в котором находится основная логика.

Допустим, мы хотим упростить добавление нового студента. Процедура будет принимать имя, фамилию и идентификатор группы, а затем вставлять новую запись:
```
CREATE PROCEDURE add_student(
    p_first_name VARCHAR(20),
    p_last_name VARCHAR(20),
    p_group_id INTEGER
)
LANGUAGE plpgsql
AS $$
BEGIN
    INSERT INTO students (first_name, last_name, group_id)
    VALUES (p_first_name, p_last_name, p_group_id);
END;
$$;
```

Использование процедуры:
```
CALL add_student('Иван', 'Волков', 1);
```

### Управление процедурами

Просмотр списка процедур:
```
SELECT proname, prosrc
FROM pg_proc
WHERE pronamespace = 'public'::regnamespace AND prokind = 'p';
```

Просмотр списка процедур через `psql`:
```
\df public.*
                                                                  Список функций
 Схема  |     Имя     | Тип данных результата |                                   Типы данных аргументов                                   |  Тип
--------+-------------+-----------------------+--------------------------------------------------------------------------------------------+-------
 public | add_student |                       | IN p_first_name character varying, IN p_last_name character varying, IN p_group_id integer | проц.
(1 строка)
```

Удаление процедуры:
```
DROP PROCEDURE add_student;
```

Для изменения процедуры необходимо сначала удалить её, а затем переписать. Можно сделать одним оператором - `CREATE OR REPLACE PROCEDURE`.

### Функции

Функции создаются похожим способом, но они должны что-то возвращать:
```
CREATE FUNCTION name(args)
RETURNS return_type
LANGUAGE plpgsql
AS $$
DECLARE
    -- переменные
BEGIN
    -- тело функции
    RETURN result;
END;
$$;
```

В качестве примера напишем функцию, которая по `student_id` возвращает полное имя студента:
```
CREATE FUNCTION get_full_name(p_student_id INTEGER)
RETURNS VARCHAR(41)
LANGUAGE plpgsql
AS $$
DECLARE
    v_full_name VARCHAR(41);
BEGIN
    SELECT last_name || ' ' || first_name INTO v_full_name
    FROM students
    WHERE student_id = p_student_id;

    RETURN v_full_name;
END;
$$;
```

Вызвать функции можно прямо в запросе:
```
SELECT get_full_name(1);

 get_full_name
---------------
 Иванов Иван
(1 строка)
```

Функция, реализованная выше, возвращает скаляр, функции также могут возвращать таблицы. Создадим функцию, которая возвращает список студентов с их средним баллом выше определенного порога:
```
CREATE FUNCTION get_students_by_grade(p_min_grade NUMERIC)
RETURNS TABLE (
    student_id INTEGER,
    first_name VARCHAR(20),
    last_name VARCHAR(20),
    avg_grade NUMERIC(3,2)
) AS $$
BEGIN
    RETURN QUERY
    SELECT s.student_id, s.first_name, s.last_name, s.average_grade
    FROM students AS s
    WHERE s.average_grade >= p_min_grade
    ORDER BY s.average_grade DESC;
END;
$$
LANGUAGE plpgsql;
```

## Триггеры

Триггер - это разновидность хранимой процедуры, которая не вызывается пользователем, а запускается автоматически сервером базы данных при наступлении определенного события (вставки, обновления или удаления).

Триггер можно представить как некое правило, например: "Если кто-то попытается удалить студента из таблицы, сначала выполни определённую функцию".

### Создание триггера

Создание триггера происходит в два этапа:
- Создание триггерной функции, которая вызывается триггером.
- Создание триггера, который вызывает функцию при определенных событиях в таблице.

Триггерная функция имеет некоторые отличительные особенности от обычной функции:
- Она не принимает аргументы.
- Она обязана возвращать тип `TRIGGER`.
- Внутри триггерной функции доступны специальные переменные: `NEW` (новая версия строки для `INSERT` и `UPDATE`) и `OLD` (старая версия строки для `UPDATE` и `DELETE`).

Создание триггера имеет следующий синтаксис:
```
CREATE TRIGGER trigger_name
    {BEFORE | AFTER | INSTEAD OF} {INSERT | UPDATE | DELETE | TRUNCATE} ON table_name
    [FOR [EACH] {ROW | STATEMENT}]
    [WHEN (condition)]
    EXECUTE FUNCTION func_name();
```

Ключевые элементы:
- `{BEFORE | AFTER | INSTEAD OF}` - в какой момент срабатывает: до, после или вместо.
- `{INSERT | UPDATE | DELETE | TRUNCATE}` - событие, на которое срабатывает триггер. Можно указать несколько, используя `OR`.
- `[FOR [EACH] {ROW | STATEMENT}]`:
  - `FOR EACH ROW` - функция вызывается для каждой изменяемой строки.
  - `FOR EACH STATEMENT` (по умолчанию) - функция вызывается один раз для события.
- `[WHEN (condition)]` - дополнительное условие срабатывания триггера.

Рассмотрим пример логгирования изменения фамилии студента. Создадим отдельную таблицу, в которую будем записывать события:
```
CREATE TABLE students_log(
    log_id SERIAL PRIMARY KEY,
    student_id INTEGER,
    old_last_name VARCHAR(20),
    new_last_name VARCHAR(20),
    changed_by VARCHAR(50),
    changed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

Триггерная функция:
```
CREATE FUNCTION log_student_lastname_change()
RETURNS TRIGGER
LANGUAGE plpgsql
AS $$
BEGIN
    IF OLD.last_name IS DISTINCT FROM NEW.last_name THEN
        INSERT INTO students_log(
            student_id, 
            old_last_name, 
            new_last_name, 
            changed_by
        ) VALUES (
            OLD.student_id,
            OLD.last_name,
            NEW.last_name,
            current_user
        );
    END IF;
    
    RETURN NEW;
END;
$$;
```

Триггер:
```
CREATE TRIGGER trigger_student_lastname_log
    AFTER UPDATE OF last_name ON students
    FOR EACH ROW
    EXECUTE FUNCTION log_student_lastname_change();
```

При обновлении фамилии студента появится запись в таблице `students_log`:
```
UPDATE students
	SET last_name = 'Васильева'
	WHERE student_id = 2;

SELECT *
FROM students_log;

 log_id | student_id | old_last_name | new_last_name | changed_by |         changed_at
--------+------------+---------------+---------------+------------+----------------------------
      1 |          2 | Васильева     | Кузнецова     | postgres   | 2026-02-28 20:50:37.715992
(1 строка)
```

### Управление триггерами

Просмотреть триггеры можно в psql:
```
\dS students
...
Триггеры:
    trigger_student_lastname_log AFTER UPDATE OF last_name ON students FOR EACH ROW EXECUTE FUNCTION log_student_lastname_change()
```

Удаление триггера:
```
DROP TRIGGER trigger_student_lastname_log ON students;
```

Можно временно отключить триггер:
```
ALTER TABLE students DISABLE TRIGGER trigger_student_lastname_log;
```

А затем включить:
```
ALTER TABLE students ENABLE TRIGGER trigger_student_lastname_log;
```


Для того, чтобы изменить триггер необходимо его сначала удалить, а затем заново создать. Можно сделать одним оператором - `CREATE OR REPLACE TRIGGER`.
