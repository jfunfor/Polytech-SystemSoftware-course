# Лекция 17 - Работа с удалёнными репозиториями

Удалённые репозитории - это версии проекта, хранящиеся на удалённом сервере. Доступ к репозиторию может осуществляться по интернету или по локальной сети. В данной лекции в качестве удалённого репозитория мы будем использовать GitHub.

## Добавление удалённого репозитория

Для просмотра списка добавленных удалённых репозиториев воспользуемся следующей командой:
```
$ git remote
```

Пустой вывод говорит о том, что нет подключенных удаленных репозиториев. Для того чтобы добавить удалённый репозиторий и присвоить ему имя (shortname), выполним команду `git remote add`:
```
$ git remote add git_test https://github.com/<git_username>/git_test

$ git remote
git_test
```

В данном случае мы подключаем удаленный репозиторий с адресом `https://github.com/<git_username>/git_test` и назначаем ему имя `git_test`.

## Клонирование репозитория

Клонирование - процесс создания локальной копии удаленного репозитория. Для клонирования репозитория необходимо воспользоваться командой `git clone` с указанием URL репозитория:
```
$ git clone https://github.com/<git_username>/git_test
Cloning into 'git_test'...
remote: Enumerating objects: 5, done.
remote: Counting objects: 100% (5/5), done.
remote: Compressing objects: 100% (4/4), done.
remote: Total 5 (delta 0), reused 0 (delta 0), pack-reused 0 (from 0)
Receiving objects: 100% (5/5), done.
```
Команда `clone` автоматически добавляет этот удаленный репозиторий под именем origin. Давайте исправим это:
```
$ git remote rename origin git_test
Renaming remote references: 100% (3/3), done.
```
Клонирование позволяет работать с уже существующим репозиторием. Однако если вы хотите вносить изменения в проект, в котором у вас нет прав на запись, или планируете развивать свою версию независимо от исходной, используйте ответвление (fork). **Fork** - это копия репозитория, которая позволяет работать над проектом независимо от оригинала. Связь между локальным репозиторием (или форком) и исходным проектом осуществляется через **upstream** - удаленный репозиторий-оригинал, из которого можно получать обновления в свой локальный репозиторий.

## Ветвление

### Удаление локальной ветки

Для локального удаления веток используется уже знакомая нам команда `git branch` с флагами `-d` и `-D`. Разница флагов заключается в том, что `-d` выполняет безопасное удаление, т. е. если в ветке есть изменения, которые могут быть утеряны, то удаление не произойдёт, пока не будет произведено слияние. Флаг `-D` предназначен для принудительного удаления. Пример удаления локальной ветки:
```
$ git branch
  add_something_new
* main

$ git branch -d add_something_new
error: the branch 'add_something_new' is not fully merged
hint: If you are sure you want to delete it, run 'git branch -D add_something_new'
hint: Disable this message with "git config set advice.forceDeleteBranch false"

$ git branch -D add_something_new
Deleted branch add_something_new (was 115c8c6).

$ git branch
* main
```

В данном примере команда `git branch` с флагом `-d`, т. е. безопасное удаление, не произвела удаление, т. к. ветка не была полностью слита, а команда с флагом `-D` обходит проверку и удаляет ветку.

### Отправка ветки в удаленный репозиторий

Команда `git push` отправляет изменения из локального репозитория в удаленный:
```
$ git push git_test feature
```

Команда выше отправляет локальную ветку `feature` в удаленный репозиторий `git_test`.

### Удаление ветки в удаленном репозитории

Для удаления ветки в удаленном репозитории также используется команда `git push` с флагом `--delete`:
```
$ git push git_test --delete feature
To https://github.com/<git_username>/git_test
 - [deleted]         feature
```

## Получение изменений из удалённого репозитория

### fetch

Для получения данных из удаленного репозитория, следует выполнить команду `git fetch`:
```
$ git fetch git_test
```

Важно отметить, что команда `fetch` забирает данные в локальный репозиторий, но не модифицирует, то над чем производились работы в локальном репозитории.

### Pull

Для автоматического получения изменений из удаленного репозитория и слияния с текущей веткой можно воспользоваться командой `git pull`:
```
$ git pull git_test main
```
Команда `git pull` извлекает данные с сервера и автоматически пытается слить (merge) их с кодом, над которым велись работы локально.
